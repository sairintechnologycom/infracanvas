## Decisions Made

- **Inline test seed helper instead of conftest fixture.** Plan suggested a project-wide `seed_share_link` fixture in `conftest.py`, but the only consumer is `test_share.py`. A module-local `_seed_share_link_row` async helper keeps the surface area smaller and avoids touching shared infrastructure other plans depend on.
- **Synthesize bcrypt-shaped placeholders for seeded rows.** Seeded share-links never go through the verify path (we only `LIST` them), so producing real bcrypt hashes would waste ~200ms/row. Strings shaped like `$2b$12$seed{hex}` satisfy column NOT-NULL constraints and uniqueness without the cost.
- **DELETE proxy uses raw fetch, not backendFetch.** `backendFetch` calls `res.json()` unconditionally. The backend returns 204 with an empty body on revoke. Calling `res.json()` on that throws `SyntaxError: Unexpected end of JSON input`. The proxy now uses `auth()` + `fetch` directly (same Bearer auth, same env-var pattern) so the 204 propagates correctly.
