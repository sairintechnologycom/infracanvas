## Decisions Made

- **Delete the dual-canvas component outright** rather than keep a feature flag. RESEARCH §Anti-Patterns explicitly warns against accumulating dead code; D-10 rejected the pattern; git history is sufficient if a future "side-by-side toggle" lands.
- **Keep DiffNodeList.tsx for now** even though CompareLayout no longer uses it — its existing tests are still exercised by the test file. Removing it without splitting tests would have inflated the diff. Recommended for cleanup in a follow-up plan.
- **Use `vi.importActual` rather than `vi.doUnmock` + `vi.resetModules`** to coexist with the existing module-level CompareLayout stub used by the RSC tests. The doUnmock approach broke the RSC stub for downstream tests; vi.importActual is scoped to the single import call.
- **`FindingsDeltaSection` body is a stub.** The Sheet drawer body is also a stub (renders a placeholder). Both stubs are documented in the components and in the threat-model dispositions. The contract RMD-02 measures the open/close UX and the section card structure — those ship.

**2. [Rule 1 — Bug] Plan's recommended `vi.doUnmock + vi.resetModules` pattern broke a downstream test**
- **Found during:** Task 3 (full test run)
- **Issue:** Module-level `vi.mock('@/components/compare/CompareLayout', ...)` (used by the RSC-validation tests) was being permanently torn down by `vi.resetModules()` calls in the new 4-section suite. The RSC test "renders CompareLayout on success" lost its stub and failed.
- **Fix:** Replaced `vi.doUnmock + vi.resetModules + import()` with a single `await vi.importActual<...>('@/components/compare/CompareLayout')` per test, which bypasses the mock for that one call without disturbing module state for other tests.
- **Files modified:** `dashboard/__tests__/compare-layout.test.tsx`
- **Verification:** All 30 tests in compare-layout.test.tsx pass; isolated `npm test -- --run __tests__/compare-layout.test.tsx` → 30/30 green.
- **Committed in:** `9787871`
