## Decisions Made

(See frontmatter `key-decisions` for the full list — repeated here as prose for the record.)

- **Two sibling helpers, not one.** Route INSERTs in `ready` (CLI commit, no UPDATE step needed); worker INSERTed at enqueue in `pending` (needs UPDATE+meter atomic). A single helper would have either back-compat-broken the route or made the worker race-prone. Two helpers, one shared `record_scan_meter_event` call site, both invariant-preserving.
- **Lazy `from app.queue.tasks.scan_repo import scan_repo` is at module-level inside the route function (NOT wrapped in a sub-function).** This keeps `scan_repo` accessible as a local binding for tests to monkeypatch. Tests instead patch `sys.modules['app.queue.tasks.scan_repo']` BEFORE the route runs the import — the import then resolves to the stub. In production sys.modules caches the import after the first request so the runtime cost is one stat call per process.
- **`type: ignore[import-not-found,import-untyped,unused-ignore]` tri-code on the lazy import.** mypy --strict reports both `import-untyped` (parent package exists, child has no py.typed) AND `import-not-found` (Plan 06 module doesn't exist), with `unused-ignore` to silence whichever doesn't fire on a given mypy run. Listing all three keeps the line clean before AND after Plan 06 lands.
- **`r2_key = ''` (empty string) for the pending INSERT, NOT NULL or a sentinel.** NULL would force a migration to make the column nullable; sentinel (e.g. `pending:{scan_id}`) would tempt routes to GET-presign on a fake key. Empty string is the existing default-equivalent and the conditional URL signing in get_scan (`status==ready and r2_key`) gates it cleanly.
- **`branch_not_found:{repo}@{branch}` error detail.** Includes the user-facing tuple so the dashboard can render a precise toast without log inspection. Acceptable info-leak — the team already owns this installation per the membership probe; learning a branch doesn't exist on a repo they have access to is non-sensitive.
- **Broad `except Exception as e` on the kiq() try block.** Orphan pending rows are worse than swallowing a specific broker error type. Even a TypeError from the lazy import (if Plan 06 lands a wrong-shaped scan_repo) flips the row to failed rather than orphan it. `# noqa: BLE001` documents the deliberate breadth.
- **Membership probe + INSERT use raw SQL `text(...)` rather than ORM.** Keeps the route under 200 LoC; avoids ORM lazy-loading subtleties on the github_* projection. Same precedent as Plan 04's install-callback INSERT.
- **Extended `ScanGetResp` keeps `presigned_get_url: str | None` (relaxed, was `str`).** Pending and failed rows return null. Plan 11's polling page uses null-vs-non-null as the "still working / show viewer" discriminant.
