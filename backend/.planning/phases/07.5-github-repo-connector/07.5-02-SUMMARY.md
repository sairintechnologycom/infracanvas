## Decisions Made

- **Two atomic migrations (007 + 008) rather than one combined migration.** Reversibility is per-migration; splitting lets a future revert touch only the scans columns without disturbing the github_installations table (or vice versa). Cost is two `Running upgrade` log lines on every fresh DB; benefit is sharper rollback granularity.
- **Partial index predicate `WHERE status='ready'` (not `status IS NOT NULL`).** Only successful scans are dedup candidates; pending/failed re-scans should still proceed. Index size stays small (only the success rows of one team contribute).
- **Index column order `(team_id, github_repo, github_sha, created_at DESC)`.** team_id leading so the planner uses the RLS predicate first; created_at DESC enables the dedup query's `ORDER BY created_at DESC LIMIT 1` to be index-only.
- **RSA fixture session-scoped (not module-scoped).** Generation cost is non-trivial (~150 ms on M-class CPU) and the key is functionally immutable; sharing across the test session is safe because tests never mutate it.
- **`gh_settings_patched` monkeypatches the live Settings singleton.** Plan 03's auth helpers import `from app.settings import settings` at module load — patching the singleton is the only way to make the test value visible without re-importing the auth module per test.
- **Smoke test for respx uses sync `httpx.Client` not `AsyncClient`.** Integration is at the transport level; sync vs async makes no difference for verifying the mock fires. Async path is exercised in downstream tests where the production code is async.
- **Test package `__init__.py` files have one-line docstrings (not empty).** Keeps `grep` + IDE 'find module' helpful; matches the existing `backend/tests/` pattern (no empty `__init__.py` exists in the repo today).
- **VALIDATION.md flag flip lives in the same commit as the smoke test.** The flag is a live consequence of fixture availability, not a separate documentation gesture; combining them keeps the audit trail honest.

## Issues Encountered

- **`pip install -e ".[dev]"` and direct `python` invocations require `source backend/.venv/bin/activate`** (host Python is 3.11; project requires 3.12). Pre-existing constraint from Plan 01; flagged here for future executors.
- **Per-module coverage gate fires on scoped pytest runs** (e.g. `pytest tests/integrations/github/test_fixtures_smoke.py`) because no `app/*` modules are touched. Use `--no-cov` for fixture-scope smoke verification; the gate is meaningful only for full-suite runs.
- **Alembic env.py requires `DATABASE_URL` to use the asyncpg driver** even for sync introspection — the `psycopg2`-driver URL fails with `InvalidRequestError: The asyncio extension requires an async driver`. Solution: always pass `DATABASE_URL=postgresql+asyncpg://...`. (Sync introspection in tests uses `create_engine` directly with a separate psycopg2 URL, avoiding env.py.)
