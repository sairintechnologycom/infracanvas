## Decisions Made

- **Empty-string defaults for `github_app_*` fields, not `None`.** Existing Settings string fields use the same convention (`stripe_meter_event_name`, `git_sha`); switching to `Optional[str]` here would force every downstream consumer (`mint_app_jwt` in Plan 03) to coerce `None to error` anyway. Empty string is the natural "unprovisioned" sentinel for env-var-derived secrets.
- **Conftest stubs intentionally set `GITHUB_APP_PRIVATE_KEY` to empty string (not a real PEM).** No test in the current repo imports `app.integrations.github.auth.mint_app_jwt` at module load, so the empty key never reaches a JWT signer at import time. Plan 02 will generate ephemeral RSA keys per-test in `tests/integrations/github/conftest.py` and monkeypatch the setting just for the auth test module — the smaller blast radius keeps the global stub harmless.
- **TDD split: RED commit = test only, GREEN commit = both impl files together.** The third test (`test_conftest_stubs_provide_github_app_defaults`) requires BOTH the Settings field AND the conftest env-stub to pass — combining them in GREEN keeps the RED to GREEN transition atomic for the whole test class.
