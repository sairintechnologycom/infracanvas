<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Package author ‚Üí Next.js consumer | Library artifact (`dist/lib/*`) is what the dashboard imports; anything in `dist/lib/` becomes part of the consumer's bundle. Source paths and internal modules must not leak. |
| Build-time Tailwind ‚Üí Runtime | Tailwind v4 CLI runs ONCE at build time producing `dist/lib/styles.css`. Runtime must not regenerate Tailwind on consumer (would DoS their SSR). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering / Repudiation | `dist/lib/index.js` entry ‚Äî `'use client'` directive at line 1 | mitigate | `vite.config.lib.ts` emits the directive via `rollupOptions.output.banner` (NOT source-level bare string ‚Äî Rollup strips those). Plan 03 CI asserts `head -1 dist/lib/index.js \| grep -q "use client"`. |
| T-05-02 | Information Disclosure | `viewer/package.json` `files` field ‚Äî controls which paths ship with the package | mitigate | `files: ["dist/lib"]` explicitly scopes publish-surface; `viewer/src/**` and `__tests__/**` cannot leak into a workspace-linked consumer.
