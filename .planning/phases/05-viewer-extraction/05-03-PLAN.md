PLACEHOLDER = "window.__INFRACANVAS_DATA__ = null;"

def export_html(graph: ResourceGraph, output_path: Path, gate_mode: bool = True) -> None:
    template = TEMPLATE_PATH.read_text()
    graph_json = graph.model_dump_json()
    gate_js = "true" if gate_mode else "false"

```yaml
name: Viewer Peer Compatibility

on:
  push:
    branches: [main]
    paths:
      - 'viewer/**'
      - 'package.json'
      - '.github/workflows/viewer-peer-compat.yml'
  pull_request:
    branches: [main]
    paths:
      - 'viewer/**'
      - 'package.json'
      - '.github/workflows/viewer-peer-compat.yml'

jobs:
  peer-compat:
    name: Build + test with React ${{ matrix.react-version }}
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        react-version: ['18', '19']
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install dependencies (default peers)
        run: npm ci

      - name: Override React version
        working-directory: viewer
        run: |
          npm install \
            react@${{ matrix.react-version }} \
            react-dom@${{ matrix.react-version }} \
            @types/react@${{ matrix.react-version }} \
            @types/react-dom@${{ matrix.react-version }} \
            --legacy-peer-deps

      - name: Type check
        working-directory: viewer
        run: npx tsc --noEmit

      - name: Build
        working-directory: viewer
        run: npm run build

      - name: Assert 'use client' banner
        working-directory: viewer
        run: |
          if ! head -1 dist/lib/index.js | grep -q "^'use client'"; then
            echo "FAIL: 'use client' banner missing from dist/lib/index.js"
            head -3 dist/lib/index.js
            exit 1
          fi

      - name: Assert no private path leakage
        working-directory: viewer
        run: |
          if grep -q "/viewer/src/" dist/lib/index.js; then
            echo "FAIL: Private /viewer/src/ path leaked into library bundle"
            grep -n "/viewer/src/" dist/lib/index.js | head -5
            exit 1
          fi

      - name: Assert bundle size < 5 MB
        working-directory: viewer
        run: |
          size=$(wc -c < dist/index.html)
          echo "dist/index.html: $size bytes"
          if [ "$size" -ge 5000000 ]; then
            echo "FAIL: Single-file HTML exceeds 5 MB"
            exit 1
          fi

      - name: Run tests
        working-directory: viewer
        run: npm test -- --run
```

**Design notes (documented for future readers, not part of the YAML):**

- `fail-fast: false` — if React 18 passes and React 19 fails (or vice versa), we want to see the failure, not have the other job cancelled.
- `npm ci` at the repo root installs via npm workspaces (root `package.json` has `workspaces: ["viewer"]` from Plan 01), pulling peer deps + `react`/`react-dom` at the versions pinned in `viewer/package.json` `dependencies`.
- The `Override React version` step re-installs React at the matrix version. `--legacy-peer-deps` bypasses npm's strict peer resolution (our `peerDependencies` intentionally overlaps with `dependencies`, which npm 10 flags without this flag).
- Three load-bearing assertions run BEFORE the test step: banner, path leakage, bundle size. These are the T-05-01 / T-05-02 mitigations documented in the threat model. If any fails, the job exits before tests and surfaces the specific failure.
- `npm test -- --run` is the same command `ci.yml` uses for the existing `test-viewer` job — ensures Vitest's run mode (no watch).
- Node 22 (LTS as of April 2026) — RESEARCH.md notes `ci.yml` uses Node 20; the peer-compat job uses Node 22 to also smoke-test a newer Node. If the ci.yml Node 20 path stays green, and this workflow's Node 22 + React 18/19 paths stay green, that covers 4 effective environments.

**DO NOT run this workflow locally** — it requires GitHub Actions runners. Verification of the workflow's correctness is limited to YAML syntax and file-existence checks. The actual matrix green/red status will be observable on the first push after this commit lands.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build artifacts → CLI template consumer | `dist/index.html` is copied verbatim into `cli/infracanvas/export/viewer_template.html` and shipped with every `pip install infracanvas` release. Contents are trusted by the Python layer (only replacement target is the placeholder). |
| Build artifacts → Next.js dashboard consumer (Phase 7) | `dist/lib/index.js` + `dist/lib/styles.css` + `dist/lib/index.d.ts` are imported directly by the dashboard. Any regression in externalization, banner, or CSS would break Phase 7. |
| GHA matrix workflow → merge gate | The peer-compat workflow is the only automated signal that React 19 works. A silent failure (e.g., YAML typo that prevents the workflow from running) would defeat the D-12 CI guarantee. |
