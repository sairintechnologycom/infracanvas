    **Decision on teams-table lookup (chicken-and-egg with RLS):** `resolve_team_from_clerk_org` must look up `Team.id` by `clerk_org_id` BEFORE a `SET LOCAL app.current_team_id` can be set — we don't know the team.id until we look it up. Directly opening a permissive SELECT policy on `teams` (e.g. `USING (true)`) leaks `stripe_customer_id` across tenants and is a *new* derivative decision not in CONTEXT.md. Rejected.

    **Chosen approach (mirrors Plan 06's `scan_team_id` helper):** expose a narrow `SECURITY DEFINER` SQL function `team_by_clerk_org(p_clerk_org_id text) RETURNS teams` that returns the single matching team row. The team-scoped SELECT policy on `teams` stays strict (only rows matching `app.current_team_id`). App code NEVER directly SELECTs teams to satisfy the lookup path — it calls the function. GRANT EXECUTE to `infracanvas_app` only; REVOKE from PUBLIC. The function is stable, read-only, and returns `teams.*` — because the caller already presented a JWT that proves they hold the `clerk_org_id`, lookup is authorized by authentication, not by table-level policy.

    **Webhook INSERT path:** webhooks INSERT into `teams` BEFORE any `app.current_team_id` context exists (team doesn't exist yet). Add a permissive `teams_webhook_insert ON teams FOR INSERT WITH CHECK (true)` policy — this is acceptable because the `/v1/webhooks/clerk` endpoint is gated by Svix HMAC signature verification upstream (T-06-02), so the INSERT path is reachable only via an authenticated webhook delivery. Mutations (UPDATE/DELETE) stay strictly `current_setting`-scoped.

    ```python
    from __future__ import annotations
    import jwt
    from fastapi import Depends, HTTPException, Request, status
    from jwt import PyJWKClient, InvalidTokenError
    from pydantic import BaseModel
    import sentry_sdk
    import structlog
    from app.settings import settings

    _jwks_client = PyJWKClient(
        settings.clerk_jwks_url, cache_keys=True, lifespan=3600,
    )

    class ClerkPrincipal(BaseModel):
        user_id: str
        session_id: str
        clerk_org_id: str
        role: str
        request_id: str = ""

    def _bearer(request: Request) -> str:
        h = request.headers.get("authorization", "")
        if not h.lower().startswith("bearer "):
            raise HTTPException(status.HTTP_401_UNAUTHORIZED, "missing_bearer")
        return h.split(" ", 1)[1].strip()

    async def require_principal(request: Request) -> ClerkPrincipal:
        token = _bearer(request)
        try:
            signing_key = _jwks_client.get_signing_key_from_jwt(token).key
            claims = jwt.decode(
                token, signing_key,
                algorithms=["RS256"],                # algorithm confusion protection (T-06-06)
                issuer=settings.clerk_issuer,
                audience=None,                       # Clerk uses azp, not aud (RESEARCH § P8)
                options={"require": ["exp", "iat", "sub", "sid"]},
                leeway=10,
            )
        except InvalidTokenError as e:
            raise HTTPException(status.HTTP_401_UNAUTHORIZED, "invalid_token") from e

        if claims.get("azp") not in settings.clerk_allowed_origins:
            raise HTTPException(status.HTTP_401_UNAUTHORIZED, "azp_mismatch")
        o = claims.get("o")
        if not o or not o.get("id"):
            raise HTTPException(status.HTTP_403_FORBIDDEN, "no_active_organization")

        rid = structlog.contextvars.get_contextvars().get("request_id", "")
        principal = ClerkPrincipal(
            user_id=claims["sub"], session_id=claims["sid"],
            clerk_org_id=o["id"], role=o.get("rol", ""), request_id=rid,
        )
        structlog.contextvars.bind_contextvars(user_id=principal.user_id, clerk_org_id=principal.clerk_org_id)
        sentry_sdk.set_user({"id": principal.user_id})
        sentry_sdk.set_tag("clerk_org_id", principal.clerk_org_id)
        sentry_sdk.set_tag("request_id", rid)
        return principal

    def require_role(*allowed: str):
        async def _dep(p: ClerkPrincipal = Depends(require_principal)) -> ClerkPrincipal:
            if p.role not in allowed:
                raise HTTPException(status.HTTP_403_FORBIDDEN, "forbidden_role")
            return p
        return _dep
    ```

    **`backend/app/auth/deps.py`** — team resolver calling the SECURITY DEFINER function:
    ```python
    from __future__ import annotations
    from fastapi import Depends, HTTPException, status
    from sqlalchemy import text
    from sqlalchemy.ext.asyncio import AsyncSession
    import sentry_sdk
    import structlog
    from app.auth.clerk import ClerkPrincipal, require_principal
    from app.db.models import Team
    from app.db.session import raw_session

    async def resolve_team_from_clerk_org(
        principal: ClerkPrincipal = Depends(require_principal),
        session: AsyncSession = Depends(raw_session),
    ) -> Team:
        """Call team_by_clerk_org(text) SECURITY DEFINER function (migration 003)
        to fetch the Team row by clerk_org_id. This narrow function-exposed lookup
        keeps the teams SELECT policy restrictive (no permissive USING (true))
        while solving the chicken-and-egg problem where we need team.id before
        we can SET LOCAL app.current_team_id.

        Authorization: require_principal already validated the JWT; the caller
        demonstrably holds the clerk_org_id. The function returns only the single
        matching row.

        404 if webhook hasn't processed organization.created yet — client should retry."""
        result = await session.execute(
            text(
                "SELECT id, clerk_org_id, name, stripe_customer_id, "
                "created_at, updated_at "
                "FROM team_by_clerk_org(:org)"
            ),
            {"org": principal.clerk_org_id},
        )
        row = result.mappings().first()
        if row is None:
            raise HTTPException(status.HTTP_404_NOT_FOUND, "team_not_provisioned")
        team = Team(**dict(row))
        structlog.contextvars.bind_contextvars(team_id=str(team.id))
        sentry_sdk.set_tag("team_id", str(team.id))
        return team
    ```

    ```python
    from __future__ import annotations
    from typing import Any
    from sqlalchemy import select, update
    from sqlalchemy.dialects.postgresql import insert as pg_insert
    from sqlalchemy.ext.asyncio import AsyncSession
    import stripe
    import structlog
    from svix.webhooks import Webhook, WebhookVerificationError
    from app.db.models import Team
    from app.settings import settings
    from app.util.ids import new_uuid7

    _log = structlog.get_logger("app.webhooks")

    def _verifier() -> Webhook:
        return Webhook(settings.clerk_webhook_secret)

    async def verify_and_dispatch(body: bytes, headers: dict[str, str], session: AsyncSession) -> None:
        try:
            payload = _verifier().verify(body, headers)
        except WebhookVerificationError as e:
            raise PermissionError("bad_signature") from e

        evt_type = payload.get("type")
        data = payload.get("data") or {}
        _log.info("clerk_webhook", event_type=evt_type, clerk_org_id=data.get("id"))

        if evt_type == "organization.created":
            await _upsert_team_on_created(session, data)
        elif evt_type == "organization.updated":
            await _upsert_team_on_updated(session, data)
        elif evt_type == "organization.deleted":
            await _soft_delete_team(session, data)
        # Swallow other events silently — Clerk may send more than we subscribe to.

    async def _create_stripe_customer(clerk_org_id: str, name: str) -> str:
        """Creates stripe.Customer and returns stripe_customer_id. Idempotent via metadata filter."""
        stripe.api_key = settings.stripe_secret_key
        existing = stripe.Customer.list(limit=1, email=None,
                                         expand=[], metadata={"clerk_org_id": clerk_org_id})
        # stripe-python doesn't accept metadata in list(); fall back to search API:
        try:
            found = stripe.Customer.search(query=f"metadata['clerk_org_id']:'{clerk_org_id}'")
            if found.data:
                return found.data[0].id
        except Exception:
            pass
        cust = stripe.Customer.create(name=name, metadata={"clerk_org_id": clerk_org_id})
        return cust.id

    async def _upsert_team_on_created(session: AsyncSession, data: dict[str, Any]) -> None:
        clerk_org_id = data["id"]
        name = data.get("name") or clerk_org_id
        # Idempotency: ON CONFLICT DO NOTHING on clerk_org_id (RESEARCH § P7)
        # RLS: teams_webhook_insert policy WITH CHECK (true) permits this INSERT —
        # authorization is provided by Svix signature verification above.
        stripe_customer_id = await _create_stripe_customer(clerk_org_id, name)
        stmt = pg_insert(Team).values(
            id=new_uuid7(),
            clerk_org_id=clerk_org_id,
            name=name,
            stripe_customer_id=stripe_customer_id,
        ).on_conflict_do_nothing(index_elements=["clerk_org_id"])
        await session.execute(stmt)

    async def _upsert_team_on_updated(session: AsyncSession, data: dict[str, Any]) -> None:
        clerk_org_id = data["id"]
        name = data.get("name") or clerk_org_id
        # teams_mutate_update requires app.current_team_id — set it to the matching team.
        # Use team_by_clerk_org helper to resolve id without a permissive SELECT policy.
        row = (await session.execute(
            __import__("sqlalchemy").text(
                "SELECT (team_by_clerk_org(:org)).id AS id"
            ), {"org": clerk_org_id}
        )).first()
        if row is None or row.id is None:
            _log.warning("org_updated_team_missing", clerk_org_id=clerk_org_id)
            return
        await session.execute(
            __import__("sqlalchemy").text("SET LOCAL app.current_team_id = :t"),
            {"t": str(row.id)},
        )
        stmt = update(Team).where(Team.clerk_org_id == clerk_org_id).values(name=name)
        await session.execute(stmt)

    async def _soft_delete_team(session: AsyncSession, data: dict[str, Any]) -> None:
        clerk_org_id = data["id"]
        row = (await session.execute(
            __import__("sqlalchemy").text(
                "SELECT (team_by_clerk_org(:org)).id AS id"
            ), {"org": clerk_org_id}
        )).first()
        if row is None or row.id is None:
            return
        await session.execute(
            __import__("sqlalchemy").text("SET LOCAL app.current_team_id = :t"),
            {"t": str(row.id)},
        )
        # Phase 6: mark name only; hard-delete deferred to avoid orphan scans
        stmt = update(Team).where(Team.clerk_org_id == clerk_org_id).values(name="[deleted]")
        await session.execute(stmt)
    ```

| T-06-02 | Tampering | forged Clerk webhook | mitigate | `svix.webhooks.Webhook.verify(raw_body, headers)`; bad signature → 401. Svix library handles timestamp skew (±5min) + timing-safe HMAC compare. | T-06-06 | Spoofing | JWT algorithm confusion | mitigate | `algorithms=["RS256"]` explicit (no `"none"` or HS256); JWKS cached with 1h lifespan; PyJWKClient rate-limits refresh. `audience=None` + separate `azp` allowlist check prevents aud-spoofing. |
