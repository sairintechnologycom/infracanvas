# Phase 7.1 Plan 09: Sparkline tooltip + voice-rule dates Summary

Closed two RMD-06 sub-findings with a TDD pass: (1) Sparkline now renders a hover tooltip on `mousemove` that picks the closest data point and shows `Score N · Apr 28` in `bg-slate-900 text-white text-xs px-2 py-1 rounded-sm` per UI-SPEC §/. (2) `RecentScansTable.tsx` swapped its compact `2h ago / 1d ago / ISO` formatter for an exported `formatRelativeDate(iso, now?)` honouring the UI-SPEC §"Voice rules" full-word form (`Just now / 1 hour ago / 2 hours ago / Yesterday / Apr 22`).

## What Shipped

**Sparkline hover tooltip** (`dashboard/components/scans/Sparkline.tsx`):
- `'use client'` retained.
- New required prop `dates: string[]` (paired with `scores`).
- `useState<number | null>` for `hoverIdx`; `useRef<SVGSVGElement>` for cursor-mapping math.
- `onMouseMove` maps cursor X to the SVG viewBox coordinate space (using `getBoundingClientRect()`) and selects the index with the smallest |Δx|.
- `onMouseLeave` clears `hoverIdx`.
- Tooltip renders inside a `relative` wrapper `<div>` (so `top: -28` floats above the polyline), absolutely-positioned at `(ptArr[hoverIdx][0] / W) * 100%`. Classes: `bg-slate-900 text-white text-xs px-2 py-1 rounded-sm pointer-events-none whitespace-nowrap`.
- Highlight `<circle/>` at the hovered point (`fill-slate-900`) so the user can see which data-point is read.
- Date format inside tooltip via `new Date(iso).toLocaleDateString('en-US', { month: 'short', day: 'numeric' })` → `Apr 28`.

**Threaded dates through caller** (`dashboard/components/home/ScoreSparkline.tsx`):
- Replaced "extract scores then reverse" with "pair `(score, created_at)`, filter to numeric scores, take last 10, reverse" so the date array stays index-aligned with the score array. Passed `dates={dates}` alongside `scores={points}`.

**Voice-rule relative-date helper** (`dashboard/components/home/RecentScansTable.tsx`):
- Replaced `formatRelativeOrAbsolute(iso)` (compact: `${hours}h ago` / `${days}d ago` / ISO `YYYY-MM-DD`) with exported `formatRelativeDate(iso, now?)` (full words).
- `< 1 hour` → `Just now`; `1 hour ago` (singular); `2..23 hours ago` (plural); 1 calendar-day diff → `Yesterday`; older → `Apr 22` (Intl en-US `month: short day: numeric`).
- Calendar-day diff uses midnight normalisation (`setHours(0,0,0,0)`) so the boundary doesn't depend on time-of-day.
- Date cell now calls `formatRelativeDate(scan.created_at)`.

**Tests added** (TDD red→green for each task):
- `__tests__/sparkline.test.tsx` (4 cases): svg present, mousemove → tooltip with `Score N · ` regex, mouseleave hides, tooltip carries the spec'd Tailwind classes.
- `__tests__/relative-dates.test.tsx` (5 cases): each voice-rule branch with `vi.useFakeTimers()` + `vi.setSystemTime(NOW)` so the assertions are deterministic.

## Deviations from Plan

### Rule 3 (auto-fix blocking issues)

**1. Sparkline `dates: string[]` prop required threading through `ScoreSparkline`**
- **Found during:** Task 1 GREEN — adding `dates: string[]` to `Sparkline` props broke the existing call in `ScoreSparkline.tsx` (TS error: missing required prop).
- **Fix:** Updated `ScoreSparkline` to pair `(score, created_at)` per scan instead of mapping `scores` alone, then pass `dates={dates}` alongside `scores={points}`. Order is preserved by `slice(0, 10).reverse()` applied to the paired array.
- **Why required (not optional):** an optional `dates?: string[]` would let the tooltip render `Score 87 · ` (with the date half blank) at any caller that forgot to wire it — silently degrading UI-SPEC compliance. Required prop forces every Sparkline consumer to make the date wiring explicit. Currently `ScoreSparkline` is the only consumer.
- **Files:** `dashboard/components/home/ScoreSparkline.tsx`
- **Commit:** `85c92bb` (bundled with the Sparkline change as one feat commit)

### Plan-author calibration

The plan's tooltip spec called out `pointer-events-none` but did not anticipate that an unwrapped tooltip (no `whitespace-nowrap`) would wrap the literal `Score 87 · Apr 28` at narrow widths and visually clip the `Apr 28` half. Added `whitespace-nowrap` defensively so the tooltip stays readable at the narrowest sparkline width. Not flagged as a deviation because the spec classes (`bg-slate-900 text-white text-xs px-2 py-1 rounded-sm`) are all preserved verbatim and pass the test's regex.
