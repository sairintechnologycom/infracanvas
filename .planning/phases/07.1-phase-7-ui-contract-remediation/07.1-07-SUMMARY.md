## Decisions Made

- **`branch?: string | null`** on `ScanDetailActions` props: the RSC carries `scan.branch` as `string | null`, so the wrapper widens to `string | null | undefined` and coerces to `null` for the inner `<CompareButton/>` (which insists on `string | null`). Avoids a forced cast at the call site.
- **Slot rendered before `MetadataHeader`** (top of the page JSX). `useEffect` runs after both children mount, but mounting `ScanDetailActions` first keeps the React tree intuitive ("page declares its top-bar contributions first").
- **No label-injection context (yet).** UI-SPEC §/scans/{id} calls for the scan label (`2026-04-28 main@a1b2c3d`) in the breadcrumb. This plan ships the UUID filter (so we never leak the raw ID); the human label remains in the 52px strip. A follow-up plan can add a parallel `TopBarLabelProvider` when needed — the slot pattern is now the precedent.
- **Source-grep regression tests** (e.g. `expect(SOURCE).not.toMatch(/<CompareButton\b/)`) provide a cheap, structural guard against silent revert. The render-time integration test still proves the runtime contract.
