## Decisions Made

- **Restored isUUID() helper:** shadcn init's `npx shadcn add` overwrote the entire `lib/utils.ts` (per shadcn convention — `cn()` lives there). The pre-shadcn `isUUID()` helper is used by `app/(dashboard)/scans/compare/page.tsx` to validate `a` and `b` searchParams (T-07-08-01 mitigation). Restored alongside `cn()` so both are available; no caller signature change. Recorded as Rule 1 deviation below.
- **Dark-mode OKLCH vars kept:** dashboard is light-only today, but the `.dark { ... }` block ships from shadcn at zero cost and prevents future drift if a toggle lands. Non-breaking.
- **`@import "shadcn/tailwind.css"` retained:** shadcn v4 emits this on top of `tw-animate-css` (it adds data-* custom variants + accordion keyframes from the `shadcn` npm package's `./tailwind.css` export). Plan was written before this v4 emission existed; verified the package export path resolves at `node_modules/shadcn/dist/tailwind.css`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Restored `isUUID()` after shadcn init overwrote `lib/utils.ts`**

- **Found during:** Task 2 commit prep (file diff showed entire pre-existing file replaced)
- **Issue:** `npx shadcn init`/`shadcn add` rewrote `dashboard/lib/utils.ts` to contain only `cn()` — the pre-existing `isUUID()` helper (RFC 4122 regex matcher used by `app/(dashboard)/scans/compare/page.tsx` to validate `a`/`b` searchParams before backend calls — T-07-08-01 mitigation) was deleted.
- **Fix:** Re-added `isUUID()` (and the `UUID_RE` regex constant) to `lib/utils.ts` alongside the new `cn()` export. Same module path, same signature → no caller changes required.
- **Files modified:** `dashboard/lib/utils.ts`
- **Verification:** `grep -n "isUUID\|export function cn" dashboard/lib/utils.ts` shows both exports; full test suite passes (compare-page tests included); `import { isUUID } from '@/lib/utils'` in `app/(dashboard)/scans/compare/page.tsx` resolves.
- **Committed in:** `a40d2fe` (Task 2)

---

**Total deviations:** 1 auto-fixed (Rule 1 — bug)
**Impact on plan:** Restoring `isUUID()` is a security correctness requirement (T-07-08-01 mitigation). No scope creep — the function predates this plan and was inadvertently removed by the shadcn CLI; restoration preserves the invariant that user-controlled UUID strings are validated before flowing into backend URLs.
