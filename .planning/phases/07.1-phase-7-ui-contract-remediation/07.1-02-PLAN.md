<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Clipboard API | `navigator.clipboard.writeText` may reject on permission denial |
| Toast string → DOM | sonner renders strings into the DOM via React text nodes (escaped) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07.1-03 | Tampering | Toast XSS via attacker-controlled error message | mitigate | sonner escapes string content by default. We only pass static strings (`'Link copied to clipboard'`, `"Couldn't copy. Copy manually instead."`) — no user input concatenated. Test asserts the literal string. |
| T-07.1-04 | Information disclosure | Clipboard write fails silently — user thinks the link copied but didn't | mitigate | Replace empty catch with `toast.error(...)` so the user sees the failure. Test #5 asserts the error toast fires. |
</threat_model>
