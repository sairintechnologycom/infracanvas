## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | branch / commit_sha / source on POST commit | mitigate | Pydantic `_STRICT` ConfigDict (`extra='forbid'`, `strict=True`) on `ScanCommitReq`; Field max_length caps (255 / 40 / 32) prevent oversized values |
| T-07-01-02 | Information Disclosure | branch / commit_sha values may include sensitive ref names | accept | Branches/SHAs are intentionally team-visible per RLS; team-scoped read policy already in place from migration 002 |
| T-07-01-03 | Denial of Service | malicious giant `source` value to bloat row | mitigate | `Field(max_length=32)` and `String(32)` column at the DB layer reject oversized writes |
| T-07-01-04 | Elevation of Privilege | downgrade() drops columns and could remove constraints | accept | Downgrade only runs on explicit admin invocation; no API path triggers it |
| T-07-01-05 | Spoofing | client lies about branch/commit_sha to mislead reviewers | accept | Phase 7 trusts CLI/webhook to populate truthfully; Phase 7.5 GitHub webhook will sign & verify |
</threat_model>
