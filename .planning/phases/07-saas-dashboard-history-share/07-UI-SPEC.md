# Phase 7 — UI Design Contract

> Visual and interaction contract for the SaaS Dashboard + Scan History + Share Links phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Covers requirements DSH-02..06, HST-01..03, SHR-01..02. Pre-populated from `07-CONTEXT.md` D-01..D-19 (locked), `05-CONTEXT.md` (viewer pkg surfaces), `06-CONTEXT.md` (backend contracts), and existing `landing/` + `viewer/` styling.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (recommended for primitives — Table, Dialog, DropdownMenu, Select, Tabs, Toast, Skeleton, Form). Initialize with `npx shadcn@latest init` in the new `dashboard/` Next 15 app at the start of the phase. |
| Preset | `new-york` style, base color `slate`, CSS variables enabled, Tailwind v4 (matches `landing/` toolchain). |
| Component library | shadcn/ui (built on Radix primitives — `@radix-ui/react-dialog`, `@radix-ui/react-dropdown-menu`, `@radix-ui/react-select`, `@radix-ui/react-tabs`). Clerk components (`<OrganizationSwitcher/>`, `<OrganizationProfile/>`, `<UserButton/>`) embedded inside shadcn shell — Clerk's appearance API restyled with `slate` palette to match (D-02, D-03). |
| Icon library | `lucide-react` (already a transitive dep via shadcn; matches `viewer/` which uses `lucide-react@0.511.0`). AWS service icons stay inside the viewer canvas only via `aws-react-icons` — dashboard chrome uses lucide. |
| Font | `Inter` via `next/font/google` for sans (matches `landing/` and viewer `--font-sans`). `JetBrains Mono` via `next/font/google` for commit SHAs, scan IDs, code snippets (matches viewer `--font-mono`). |
| Sparkline | Inline handrolled SVG (~20 LOC) — no Recharts, no `react-sparklines`. Vercel bundle budget protected; sparkline is a 10-point polyline + min/max dot, no axes, no tooltips. |
| Date pickers | `react-day-picker` (shadcn `<Calendar/>` block — already shadcn-blessed, no third-party registry). Used only for the `/scans` custom date-range filter. |
| Charts beyond sparkline | None in Phase 7. Reserve slot for CostLens (Phase 9) and FlowMap (Phase 10+). |

**Visual mode:** The dashboard chrome is **light mode by default** (white surfaces, slate borders) to harmonize with the embedded `<DiagramCanvas/>` from `@infracanvas/viewer` which is light-only and ships full-bleed inside `/scans/{id}`, `/compare/{from}/{to}`, and `/share/{token}`. Landing (`landing/`) stays dark; dashboard is light. Standard marketing-dark → product-light SaaS convention. Dark mode for the dashboard is **explicitly deferred** per Phase 5 D-09/D-10 (theme tokens locked at viewer-pkg level only).

## Spacing Scale

Declared values (all multiples of 4 — Tailwind 4 default scale, no overrides):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`space-1`) | Icon-to-label gaps, table cell inline padding |
| sm | 8px (`space-2`) | Compact element spacing (filter chip gap, sidebar nav-item internal padding) |
| md | 16px (`space-4`) | Default element spacing, card internal padding, table row vertical padding |
| lg | 24px (`space-6`) | Card-to-card spacing, section padding inside main content |
| xl | 32px (`space-8`) | Layout column gaps on the home dashboard |
| 2xl | 48px (`space-12`) | Major section breaks (rare — only on the home dashboard between hero card and sparkline strip) |
| 3xl | 64px (`space-16`) | Page-level top padding under the top-bar on settings sub-pages |

**Container widths:**
- Sidebar: fixed `220px` (D-01).
- Top bar: fixed `48px` height.
- Scan-detail header strip: fixed `52px` height (single-line content).
- Compare summary strip: fixed `52px` height (single-line summary).
- Share-landing top bar: fixed `48px` height.
- Main content max-width: none on viewer pages (`/scans/{id}`, `/compare/{from}/{to}`, `/share/{token}`) — full-bleed; on `/`, `/scans`, `/settings/*` — `max-w-7xl` (1280px) centered with `px-8` (32px) gutters.

**Touch / hit targets:** Minimum `36px` for icon-only buttons (Compare, Share, Revoke). Sortable column headers expand entire `<th>` to clickable area.

**Exceptions:** Sparkline strip on `/` uses `4px` and `2px` for stroke-width (sub-token, SVG primitive). Score badge uses `28px` square (sub-token, single-component visual primitive).

## Typography

Exactly 4 sizes + 2 weights. No additional sizes introduced without checker approval.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 (21px) | Default text — table cells, form labels, finding descriptions, modal body text. Tailwind: `text-sm`. |
| Label | 12px | 600 (semibold) | 1.4 (16.8px) | Sortable column headers (uppercase + tracking-wide), filter chip labels, sidebar nav-item labels, breadcrumb separators. Tailwind: `text-xs font-semibold`. |
| Heading | 16px | 600 (semibold) | 1.4 (22.4px) | Card titles ("Latest scan", "Top critical findings", "Recent scans"), modal titles, settings sub-page titles. Tailwind: `text-base font-semibold`. |
| Display | 28px | 600 (semibold) | 1.2 (33.6px) | Score number on the latest-scan hero card (`87` next to grade `B+`). The ONLY display-sized number in the dashboard. Tailwind: `text-[28px] font-semibold`. |

**Monospace contexts** (font-family swap only — sizes inherit from above):
- Commit SHAs (always 7-char abbreviated): `font-mono text-xs` in tables, `font-mono text-sm` in scan-detail header strip and share landing top bar.
- Branch names: `font-mono text-sm` in scan-detail strip; plain sans in table cells (branches are arbitrary user input — readability beats convention).
- Scan IDs / share tokens (rare — only in URL-copy fields and revoke confirmation modals): `font-mono text-xs`.
- CLI hint in empty state: `font-mono text-sm` (`$ pip install infracanvas`).

**Numerical alignment:** All score, count, and finding numbers in tables use `tabular-nums` (Tailwind `tabular-nums`) so columns align in dense scan-list rows.

**Voice rules:**
- Sentence case for all UI strings (headings, buttons, labels). Title Case ONLY for the brand wordmark `InfraCanvas`.
- No exclamation marks anywhere in chrome (engineering audience).
- No emoji except `⚠` (for the "Never" expiry warning) and `✓` (for the copy-button confirmation flash).
- Numbers use thousand separators (`1,234 findings`) starting at `≥ 10000`; below that, no separator.
- Dates in scan-list cells: `2026-04-28` (ISO yyyy-mm-dd, locale-stable, monospace-aligned in `tabular-nums`).
- Dates in modals/strips: `Apr 28, 2026 · 14:32 UTC` (human-friendly + UTC suffix — engineers prefer UTC for build/scan timestamps).
- Relative dates only on the home-dashboard latest-scan card: `2 hours ago` / `Yesterday` / `Apr 26` (older than 7 days drops back to absolute).
- Commit SHAs: always 7-char abbreviated, `font-mono`, no leading `#` or `@`.

## Interaction & Accessibility Contracts

- **Keyboard navigation:**
  - `Tab` order: sidebar nav top-to-bottom, then top-bar action slot, then main content's first interactive (e.g. filter bar on `/scans`, `[Compare]` on `/scans/{id}`).
  - `j` / `k` (vim) on `/scans` table to move row selection up/down; `Enter` to open scan. Documented in a `kbd` hint on hover of the table title.
  - Modal `Esc` closes.
  - Compare-picker modal: `↑` / `↓` to walk through results, `Enter` to compare.
- **Focus rings:** shadcn default — `ring-2 ring-slate-400 ring-offset-2`. Never amber (amber is reserved for accent surfaces, not focus state).
- **Aria-labels for icon-only buttons:**
  - `[Copy]` (in share modal) → `aria-label="Copy share link to clipboard"`.
  - `[Revoke]` → `aria-label="Revoke share link {short-token}"`.
  - Sortable column headers → `aria-sort` set to `ascending` / `descending` / `none`.
  - Sidebar nav items → already labelled (icon + text).
- **Color contrast:** all text on backgrounds is verified ≥ 4.5:1. The amber-400 CTA on slate-900 text passes (≥ 7:1). Severity backgrounds (`bg-red-50` etc.) paired with `text-red-700` pass.
- **Loading boundaries:** every route uses Next 15 RSC + `<Suspense/>` with explicit shadcn `<Skeleton/>` fallbacks shaped to layout. No spinner anywhere except inside the viewer (which the viewer pkg owns).
- **Toasts:** shadcn `<Toaster/>` mounted at app root, positioned bottom-right, `text-sm`, max-width 360px, dismiss-on-click + auto-dismiss 3s for success / persistent for destructive.

## Responsive Targets

Per DSH-06 + PROJECT.md "modern browsers only, web-first":

- **1440p (1440 × 900 logical, ~16:10):** primary target. Layout fits without horizontal scroll; sidebar 220px + main 1220px.
- **1080p (1920 × 1080):** secondary target. Layout uses `max-w-7xl` (1280px) so 1920p just adds gutters — no layout change.
- **Below 1080p:** explicitly out of scope. No mobile breakpoints. The login page (Clerk-hosted) handles small screens by virtue of being Clerk-hosted; our authed routes do not.
- The viewer canvas fills its container regardless of viewport; it is already responsive at the canvas level (Phase 1/5).
