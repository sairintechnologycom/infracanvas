## Implementation Decisions

Each decision below is treated as a locked requirement. Plans MUST cover every D-ID. Naming convention: `D-NN` matches the priority table row in the LIVE review (1–14).

### P0 Defects (Wave 1 — Unblockers)

#### D-01 — Viewer embed layout: `App.tsx` `h-screen w-screen` → `h-full w-full`
**Decision:** In `viewer/src/App.tsx:95`, change root container className from `flex flex-col h-screen w-screen` to `flex flex-col h-full w-full`. Container hierarchy already supplies the constrained height (`flex-1 min-h-0` on the scan detail page) and width (`h-full w-full` on `ScanViewerClient`).
**Why:** `w-screen` = 100vw breaks out of the dashboard sidebar's flex container; `h-screen` = 100vh ignores the parent's constrained height. Both are artifacts of the standalone HTML export mode that the viewer was originally designed for.
**Acceptance:** `viewer/src/App.tsx` contains exactly `flex flex-col h-full w-full` (no `h-screen`, no `w-screen`); standalone HTML export still renders full-viewport (because the body/root in the standalone template IS the viewport).

#### D-02 — Singleton store leak: migrate FilterPanel / DetailPanel / SummaryBar / SearchBar (+ FlowMap variants) to `useViewerStoreOrSingleton`
**Decision:** In each of the 4 viewer components AND any FlowMap variants under `viewer/src/components/`, replace `import { useStore }` with `import { useViewerStoreOrSingleton }` and replace each `useStore(...)` call site with `useViewerStoreOrSingleton(...)`. The factory hook already exists and is correctly implemented.
**Why:** The dual-store fix from the 2026-05-01 debug session migrated `DiagramCanvas` and `ResourceNode` only — the rest were deferred. Phase 7.1's tab shell now renders all of them; with singleton bindings they read from a `graph: null` global and stay invisible.
**Acceptance:** `grep -rn "useStore" viewer/src/components/{FilterPanel,DetailPanel,SummaryBar,SearchBar}.tsx` returns 0 hits; `grep -rn "useViewerStoreOrSingleton" viewer/src/components/{FilterPanel,DetailPanel,SummaryBar,SearchBar}.tsx` returns ≥1 hit per file. Same checks pass for any `viewer/src/components/flowmap/*.tsx` files that previously called `useStore`.

#### D-03 — `/settings` 404: add `redirect('/settings/members')` page
**Decision:** Create `dashboard/app/(dashboard)/settings/page.tsx` that calls `redirect('/settings/members')` from `next/navigation`. Server component. ~3 lines.
**Why:** Settings layout currently has only sub-routes; root path returns Next.js's default bare 404, breaking the success-criteria item 3 of the phase goal.
**Acceptance:** File `dashboard/app/(dashboard)/settings/page.tsx` exists; contains `import { redirect } from 'next/navigation'` and `redirect('/settings/members')`; visiting `/settings` lands on `/settings/members` with sidebar intact.

#### D-04 — Sparkline ovals: remove `preserveAspectRatio="none"`; raise container height
**Decision:** In `dashboard/components/scans/Sparkline.tsx`, remove `preserveAspectRatio="none"` from the `<svg>` tag (let it default to `xMidYMid meet`). Keep `className="w-full"`. In `dashboard/components/home/ScoreSparkline.tsx`, change container from `h-[64px]` to `h-[80px]` minimum (target: 80px to match success-criteria item 4 wording; do NOT regress the recent fix at commit `e6135f3`).
**Why:** `preserveAspectRatio="none"` stretches r=2.5 circles into 37px ovals at 1200px×64px containers. Removing it preserves circle geometry; raising container height keeps the sparkline visually proportionate.
**Acceptance:** `grep "preserveAspectRatio" dashboard/components/scans/Sparkline.tsx` returns 0 hits OR matches `xMidYMid meet`; ScoreSparkline container height ≥ `h-[80px]`; visual: data-point dots render as circles, not ovals; existing regression test for sparkline width (added at `1281d8a`) still passes.

### P1 Defects (Wave 2 — Polish)

#### D-05 — Grade unification: extract `gradeInfo()` and adopt across ScoreCard, ScansTable, MetadataHeader, ShareViewer
**Decision:** Create a single shared util — preferred location `dashboard/lib/grade.ts` exporting `gradeInfo(score: number): { letter: string; ... }`. Canonical thresholds: A+ ≥95, A ≥90, B+ ≥85, B ≥80, C ≥70, D ≥60, F <60. Import into `ScoreCard.tsx`, `ScansTable.tsx` (`ScoreGradePill`), `MetadataHeader.tsx` (`ScoreGradePill`), and `ShareViewer.tsx`. Delete the 3 inline `ScoreGradePill` letter-derivation blocks; pill styling can stay inline but the letter MUST come from `gradeInfo()`.
**Why:** A scan scoring 82 currently shows "B" on the home hero card and "B+" in the table/header for the same scan on the same page load. Success-criteria item 5 mandates identical grade across all 4 surfaces.
**Acceptance:** `dashboard/lib/grade.ts` exists with `gradeInfo()` exported; `ScoreCard`, `ScansTable`, `MetadataHeader`, `ShareViewer` all import `gradeInfo` from it and call it for the letter; no remaining `score >= 80` literal in any of those four files outside the shared util; new unit test `dashboard/lib/grade.test.ts` covers boundary scores 95/94/90/89/85/84/80/79/70/69/60/59.

#### D-06 — Sidebar dead zone: drop `xl:w-[220px] w-12` collapse; always `w-[220px]` at desktop
**Decision:** In `dashboard/components/layout/Sidebar.tsx:41`, change className from `${visibility} md:flex xl:w-[220px] w-12 ...` to `${visibility} md:flex w-[220px] ...`. Remove `xl:inline hidden` / `xl:block hidden` qualifiers from label spans (labels always visible at md+). Keep the `hidden md:flex` pattern so genuine mobile (<768px) still relies on the TopBar hamburger.
**Why:** Spec is desktop-only (1440p / 1080p); the `xl` breakpoint creates an unintentional 768–1279px icon-only state with no expand toggle. Success-criteria item 6 explicitly forbids the `w-12` dead zone in this range.
**Acceptance:** `grep "w-12" dashboard/components/layout/Sidebar.tsx` returns 0 hits; `grep "xl:w-\[220px\]" dashboard/components/layout/Sidebar.tsx` returns 0 hits; nav labels render at 768px viewport in a manual check.

#### D-07 — TopFindings spec-vs-API mismatch: implement option (1) — add `/api/top-findings?scan_id=...` route handler
**Decision:** Resolve via **Option 1** from the LIVE review's fix sketch: add a Next.js route handler at `dashboard/app/api/top-findings/route.ts` that accepts `?scan_id=...`, fetches the scan JSON from R2 via the existing presigned URL flow (`backendFetch` server-side), returns the top 3 critical findings (rule_id, title, resource_id) sorted by severity. Update `TopFindings.tsx` to fetch from this route and render 3 finding cards per spec — `border-l-4 border-sev-critical bg-red-50/40 p-4` with rule-id badge, title, resource id, and `Open scan →` link. Reason for Option 1 over the other two: avoids a backend change; keeps the data fetch on the dashboard side; doesn't require a spec amendment.
**Why:** Spec §/ item 3 expected individual finding rows; current implementation shows only a count. This is the largest single visual regression on the home page.
**Acceptance:** Route handler exists at `dashboard/app/api/top-findings/route.ts`; returns shape `{ findings: { rule_id, title, resource_id }[] }` (max length 3); `TopFindings.tsx` no longer contains the "Per-finding details … load with the full scan graph" placeholder string; renders 3 cards each with `border-l-4 border-sev-critical bg-red-50/40 p-4`. If a scan has zero critical findings, render an empty-state line (`No critical findings — nice work.`) instead of the placeholder.

#### D-08 — ShareModal cleanup: shadcn `<Select/>`, "Link expires in", `text-amber-600` warning
**Decision:** In `dashboard/components/share/ShareModal.tsx`:
1. Replace native `<select id="share-expiry">` (lines ~139–150) with shadcn `<Select><SelectTrigger/><SelectContent>...<SelectItem value="..."/></SelectContent></Select>`. Same option set as today: 1 day / 7 days / 30 days / Never.
2. Change the label text from `"Expires in"` to `"Link expires in"` (matches spec).
3. Change the `<p>` warning className on line ~155 from `text-xs text-slate-700 mt-1` to `text-xs text-amber-600 mt-1`. Warning copy unchanged.
**Why:** Success-criteria item 7 mandates all three. Native select in a shadcn dialog also breaks visual consistency.
**Acceptance:** `grep "<select" dashboard/components/share/ShareModal.tsx` returns 0 hits; file imports `Select` from `@/components/ui/select`; label string `"Link expires in"` present; `text-amber-600` present on warning paragraph; existing ShareModal tests (test suite expanded at memory 1136) still pass with updates for new selectors if needed.

### P2 Defects (Wave 3 — Sweep)

#### D-09 — `@xyflow/react` dependency + ReactFlow CSS import in dashboard
**Decision:** Add `@xyflow/react` as an explicit dependency in `dashboard/package.json` (pin to the version already present in the root workspace `node_modules`, currently `12.6.0`). Additionally, add `import '@xyflow/react/dist/style.css'` to `dashboard/app/globals.css` (preferred) so MiniMap rendering is robust regardless of workspace hoisting.
**Why:** MiniMap renders solid black because `react-flow__minimap` background-color rule isn't loaded. Currently relies on workspace hoist to root `node_modules` which is environment-dependent.
**Acceptance:** `dashboard/package.json` `dependencies` contains `"@xyflow/react"` with a concrete version; either `dashboard/app/globals.css` contains `@import '@xyflow/react/dist/style.css'` OR a `.tsx` shell layout has the side-effect import; `npm install` succeeds in `dashboard/` without warnings; manual: MiniMap renders with light background on `/scans/[id]`.

#### D-10 — FilterPanel typography: replace `text-[9px]` / `text-[10px]` / `text-[11px]` with `text-xs`
**Decision:** In `viewer/src/components/FilterPanel.tsx` (and any other viewer component that uses sub-spec sizes), replace every `text-[9px]`, `text-[10px]`, `text-[11px]` literal with `text-xs` (12px). Update section header dividers to use `text-xs uppercase tracking-wide` consistently.
**Why:** Spec defines exactly 4 type sizes (12/14/16/28). Sub-spec sizes break the typography contract.
**Acceptance:** `grep -rn "text-\[9px\]\|text-\[10px\]\|text-\[11px\]" viewer/src/components/` returns 0 hits.

#### D-11 — Homepage `<header>` removal + `gap-12` → `gap-6`
**Decision:** In `dashboard/app/(dashboard)/page.tsx`:
1. Remove the `<header className="flex items-baseline justify-end">...</header>` block (lines ~40–44). Branch context already lives in `ScoreCard`'s right column.
2. Change the outer container className `flex flex-col gap-12` to `flex flex-col gap-6`. If the spec's "gap-12 only between hero card and sparkline strip" rhythm matters, add an explicit `mt-12` (or wrapper) on the sparkline strip alone — but default to plain `gap-6` and only re-add the larger break if the sparkline visually merges into the hero.
**Why:** Spec defines 2xl/gap-12 as rare; current homepage uses it between every section, producing 48px gaps throughout. Header element adds dead space with no content hierarchy.
**Acceptance:** `grep "<header" dashboard/app/(dashboard)/page.tsx` returns 0 hits; outer container className contains `gap-6` (not `gap-12`); homepage renders without the right-aligned "Latest scan from main" muted line.

#### D-12 — RecentScansTable + TopFindings: link colors `text-slate-900` → `text-amber-600`
**Decision:** In `dashboard/components/home/RecentScansTable.tsx` (the `View all →` link) and `dashboard/components/home/TopFindings.tsx` (the `Open scan →` link), change className from `text-sm text-slate-900 hover:text-slate-700 hover:underline` to `text-sm text-amber-600 hover:underline`.
**Why:** Spec reserves amber for navigational CTAs exactly like these. Plan-08's amber sweep over-converted these specific links the wrong direction.
**Acceptance:** Both files contain `text-amber-600` on those link elements; neither contains `text-slate-900` on those specific link spans.

### P3 Defects (Wave 3 — Sweep, bundled with P2)

#### D-13 — `ScanViewerClient` retry: `router.refresh()` not `window.location.reload()`
**Decision:** In `dashboard/components/scans/ScanViewerClient.tsx:88`, import `useRouter` from `'next/navigation'`; call `router.refresh()` from the error-state retry button instead of `window.location.reload()`.
**Why:** Spec says retry should re-fetch the same RSC; full-page reload loses sidebar state.
**Acceptance:** File contains `import { useRouter } from 'next/navigation'`; `useRouter()` called in component; retry button onClick calls `router.refresh()`; `grep "window.location.reload" dashboard/components/scans/ScanViewerClient.tsx` returns 0 hits.

#### D-14 — `/scans/compare` 404 copy: `"it."` → `"its team."`
**Decision:** In `dashboard/app/(dashboard)/scans/compare/page.tsx:64`, change the trailing copy `"you may not have access to it."` to `"you may not have access to its team."`.
**Why:** Multi-tenant context — "team" is significant copy in this product.
**Acceptance:** `grep "its team" dashboard/app/(dashboard)/scans/compare/page.tsx` returns ≥1 hit; `grep "access to it\." dashboard/app/(dashboard)/scans/compare/page.tsx` returns 0 hits.
