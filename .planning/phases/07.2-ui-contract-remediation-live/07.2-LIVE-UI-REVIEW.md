## Top 3 Next-Iteration Fixes

1. **ScoreCard "Open scan →" link uses text-slate-900 instead of text-amber-600** (`dashboard/components/home/ScoreCard.tsx:93`) — Inconsistency with the amber navigational CTA convention now applied to RecentScansTable and TopFindings. Not a task-blocker but creates a color inconsistency on the home page. Fix: change `text-xs text-slate-900` to `text-xs text-amber-600` on the ScoreCard link.

2. **DiffNodeList changed-row border uses border-amber-500 instead of spec border-amber-200** (`dashboard/components/compare/DiffNodeList.tsx:17`) — Spec §compare drift colors specifies `border-amber-200` for Changed row chrome. Current code uses `border-amber-500` (more saturated). Not a contrast failure, but deviates from the declared drift palette. Fix: change `border-l-2 border-amber-500` to `border-l-2 border-amber-200` in the `changed` style map.

3. **ScoreSparkline spec height is 120px; actual is 80px** (`dashboard/components/home/ScoreSparkline.tsx:35`) — UI-SPEC §/ specifies the sparkline strip height as `120px`; D-04 decision locked `h-[80px]` as an acceptable intermediate (the CONTEXT says "80px minimum"). The spec's declared value is 120px. The 40px shortfall makes the chart visually compact. Fix: raise container to `h-[120px]` to match spec exactly.
