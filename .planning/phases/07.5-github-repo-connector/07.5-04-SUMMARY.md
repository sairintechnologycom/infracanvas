## Decisions Made

- **CC-3 placement of `set_config` for install-callback.** The state-CSRF check happens BEFORE any team-scoped DB query, so the state-mismatch path returns 403 without ever opening a team-scoped tx. Only the upsert tx (after state passes AND the App-JWT reverify succeeds) runs `set_config('app.current_team_id', :t, true)`. This matches the CC-3 contract from PATTERNS — "team_id is known via resolve_team_from_clerk_org but not yet bound to the GUC; the state check happens FIRST."
- **Membership probe runs in its own tx, then closes — the GitHub call is OUTSIDE the tx.** `/repos` and `/branches` both open a `session.begin()` for the membership SELECT and let the tx close before awaiting the (slow, network-bound) GitHub call. Holding a Postgres connection across a multi-second GitHub API call would risk pool exhaustion under load. The downside (a tiny race window if the install gets revoked between the membership check and the GitHub call) is acceptable — GitHub's installation token mint will fail in that window with a 401 anyway, which is fine because the token mint already raises `HTTPStatusError`.
- **The `_ratelimit_to_503` helper takes the original `HTTPStatusError` argument.** Even though the function body doesn't use the arg today, the signature lets us bind richer log context later (X-RateLimit-Remaining, X-RateLimit-Reset) without breaking call sites. Mypy `--strict` is happy because the body returns an `HTTPException`. Cost: zero — the arg is captured as a local and discarded.
- **DASHBOARD_URL via `os.environ.get` with localhost fallback** rather than a Settings field. Mirrors the existing precedent in `share.py:87` (`_share_url`). Adding a Settings field would force every test conftest + .env example to define a default; env-var fallback is locally scoped and obviously skippable in tests via `monkeypatch.setenv`.
- **`test_install_callback_install_reverified_via_app_jwt` asserts the JWT shape, not the JWT contents.** Verifying the token's `iss`, `iat`, `exp` claims would couple the test to `mint_app_jwt`'s exact format (and risk false-failures on every TTL bump). Asserting the bearer header has 3 dot-separated segments + the token-mint endpoint was NOT called is sufficient to discriminate App JWT (RS256-signed, dotted) from installation token (opaque `ghs_*` string).
- **`test_install_callback_idempotent` re-mocks the respx route in between calls.** Mock state is per-test-fixture, but the mock RESPONSE can be replaced. The test calls the route once with `login="old-name"`, replaces the mock with `login="new-name"`, calls again, then asserts the DB row holds `"new-name"` (proving the UPDATE branch of `ON CONFLICT DO UPDATE` actually fired).
- **Acceptance criterion for `grep -c '@router.get(\"/install-callback\")' = 1`** — one decorator instance, not the per-segment match. Verified: 1.
