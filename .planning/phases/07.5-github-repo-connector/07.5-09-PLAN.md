   Component signature: ScanTriggerForm({ installationId: number })
   Internal state: repo (RepoResp | null), branch (string, init ""), path (string, init "."), submitting (boolean), error (string | null).
   canSubmit derived: !!repo && !!branch && !submitting.

   Render structure (vertical stack with space-y-3 + p-4 + border + rounded-md):
   - Label "Repository" + RepoCombobox (installationId, value=repo, onSelect=(r)=> setRepo(r) + setBranch("")).

   - Guard on canSubmit && repo.
   - setSubmitting(true), setError(null).

   - data = await res.json() typed as { scan_id: string }.
   - router.push("/scans/" + data.scan_id).
   - On error: setError to err.message or "Scan failed to start"; setSubmitting(false).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07.5-09-01 | DoS | Infinite poll loop on hung backend | mitigate | Poll caps at 5 attempts (~15s) and only runs when ?install=success AND empty list. After cap, render a "Click refresh" hint. Test test_poll_stops_once_list_populated. |
| T-07.5-09-02 | Tampering | Form submission to wrong installation_id | mitigate | installationId prop is sourced from server-fetched installation row (RLS-isolated by Plan 04). Backend Plan 05 step 1 re-validates installation belongs to team. |
| T-07.5-09-03 | Information Disclosure | Error messages leak backend internals | mitigate | ScanTriggerForm maps error codes (http_503, etc.) to friendly user-facing strings; never renders raw backend response body. |
</threat_model>
