# Phase 7.5 Plan 06: scan_repo Taskiq Job + r2.put_bytes Summary

**Worker-side end of the GitHub-triggered scan pipeline. `r2.put_bytes` adds an async direct-PUT helper for trusted writers; `scan_repo` is the high-risk @broker.task that mints a fresh installation token, shallow-clones the repo at HEAD, validates the user-supplied subpath against tmp_root, runs `infracanvas scan --ci` as a timeboxed subprocess, uploads the resulting JSON blob to R2 under `teams/{team_id}/scans/{scan_id}.json`, and calls Plan 05's `finalize_scan` helper to atomically flip `status='pending' -> 'ready'` and fire the Stripe meter event. Five threat classes converge here: token leakage in argv (accepted on Fly per-app VM tenancy), token leakage in stderr/structlog (mitigated via `_redact_token` chokepoint), path traversal via subpath (mitigated via `Path.resolve().relative_to()` guard), hung subprocess (mitigated via `asyncio.wait_for + proc.kill()` with 60s clone / 480s scan caps), and `status='ready'` race clobber (mitigated via `WHERE id=:id AND status='pending'` guard on the failure-path UPDATE). 16 new tests across two files; ruff + mypy --strict clean across new code. After this lands, the full backend pipeline (POST /v1/scans/from-github -> pending row -> enqueued job -> clone -> scan -> R2 -> ready) works end-to-end.**

### Task 2 — `scan_repo` taskiq job (full pipeline)

- **`backend/app/queue/tasks/scan_repo.py`** (NEW, 350 LoC) ships the @broker.task with the locked 7-kwarg signature:
  - Decorator: `@broker.task(retry_on_error=True, max_retries=3, delay=5, task_name="scan_repo")`.
  - Signature: `async def scan_repo(scan_id: str, installation_id: int, repo: str, branch: str, sha: str, path: str, team_id: str) -> None` — matches Plan 05's `kiq()` site exactly.
  - Module-level constants: `CLONE_TIMEOUT_S = 60.0`, `SCAN_TIMEOUT_S = 480.0` (8 min). Float so tests can override to 0.05s.
  - `_redact_token(text_blob, token) -> str` — chokepoint helper.
  - `_make_tmp_root() -> Path` — factory (monkeypatch hook for tests).
  - `_extract_summary(payload_bytes) -> dict | None` — pulls the `summary` key without re-validating the full ResourceGraph (that's the indexing job's responsibility).
- **Body structure (8 sections, in order):**
  1. **Bind structlog + Sentry context** at job entry (scan_id, team_id, installation_id, repo, branch, sha[:8]). Sentry tags applied via `sentry_sdk.set_tag` for the same fields.
  2. **Mint installation token** via `mint_installation_token(installation_id)` (Plan 03 helper).
  3. **mkdir tmp_root** with `exist_ok=False` (uuid4 collision is astronomical; failing loud if it ever happens beats writing into a foreign dir).
  4. **Shallow clone** via `asyncio.create_subprocess_exec("git", "clone", "--depth=1", "--single-branch", "--branch", branch, clone_url, str(tmp_root), env={**os.environ, "GIT_TERMINAL_PROMPT": "0"}, stdout/stderr=PIPE)` wrapped in `await asyncio.wait_for(clone_proc.communicate(), timeout=CLONE_TIMEOUT_S)`. On `TimeoutError` -> `clone_proc.kill() + await clone_proc.wait() + raise RuntimeError("git clone timed out after Ns")`. On non-zero rc -> sanitize stderr via `_redact_token` then `raise RuntimeError("git clone failed (rc=N): ...")`.
  5. **Path-traversal guard:** `candidate = (tmp_root / path).resolve(); candidate.relative_to(tmp_root.resolve())` — raises ValueError on escape (catches both `../` and absolute paths in one check). `is_dir()` check rejects nonexistent subpaths.
  6. **Run scan** via `asyncio.create_subprocess_exec("infracanvas", "scan", ".", "--ci", "--output", str(scan_json_path), cwd=str(candidate), env={...PYTHONUNBUFFERED=1...}, stdout/stderr=PIPE)` with same `wait_for + kill` timeout pattern at `SCAN_TIMEOUT_S`. **Critical:** `if scan_proc.returncode not in (0, 1)` — rc=0 (no findings) AND rc=1 (findings present) are BOTH success; only rc=2 is failure (landmine 13).
  7. **Read + hash + upload:** `payload_bytes = scan_json_path.read_bytes(); sha256 = hashlib.sha256(...).hexdigest(); size_bytes = len(...); summary_json = _extract_summary(...)`. R2 key is `f"teams/{team_id}/scans/{scan_id}.json"`; `await put_bytes(r2_key, payload_bytes, "application/json")`.
  8. **Finalize** via fresh team-scoped session: `set_config('app.current_team_id', :t, true)`, `SELECT id, stripe_customer_id FROM teams WHERE id=:id`, then `await finalize_scan(session, scan_id, team_id, stripe_customer_id, r2_key, sha256, size_bytes, summary_json)`. The helper does the UPDATE `pending -> ready` + Stripe meter atomically. StripeError propagates -> SmartRetryMiddleware retries the whole job.
- **Failure-path UPDATE** (outer except Exception): redact token from `str(exc)` for log line; open fresh team-scoped session; `UPDATE scans SET status='failed', error_message=:msg WHERE id=:id AND status='pending'` — pending guard prevents clobbering a row that already raced to ready (T-07.5-06-07). DB failure inside this block is itself caught (`except Exception as db_exc`) and logged-only so the original exception still re-raises for taskiq.
- **Cleanup** (finally): `if tmp_root.exists(): shutil.rmtree(tmp_root, ignore_errors=True)` — runs regardless of success or failure.

## Decisions Made

(See frontmatter `key-decisions` for the full list — repeated here as prose for the record.)

- **`_make_tmp_root()` factory.** Plan spec inlined `tmp_root = Path('/tmp') / f'scan-{uuid.uuid4()}'`. Hoisting to a helper has zero behavioral cost and gives tests a single seam to redirect cleanup to a pytest tmp_path. Eight of 13 tests rely on this.
- **Worker calls `finalize_scan`, NOT `fire_scan_meter_or_502`.** The 502-translation wrapper is route-only; the worker wants the raw StripeError to bubble out so SmartRetryMiddleware can retry. Plan objective enforced this explicitly.
- **rc=0 AND rc=1 both = success (landmine 13).** The condition `if scan_proc.returncode not in (0, 1)` is the success-discriminator; only rc=2 raises. `test_scan_rc1_treated_as_success` regression-locks this against future "tighten the success branch" refactors.
- **Token redaction defense-in-depth (3 layers):** (a) stderr decode -> `_redact_token` -> into RuntimeError message; (b) outer `except` re-applies `_redact_token` to `str(exc)` before structlog.error; (c) `bind()` at job entry never receives `token` as a kwarg (`test_token_never_in_log_bind` enforces). One miss in any layer would leak; all three together make leakage statistically improbable.
- **`stripe_customer_id` read at finalize time, NOT plumbed through kicker labels.** Customer id can change between enqueue and execution (Phase 6 D-08 customer-update flow); the worker reads it fresh from teams table. Trade-off: extra SELECT per scan vs label observability bloat. Picked SELECT.
- **`_extract_summary` does NOT re-validate against ResourceGraph.** That's the indexing job's responsibility (`enqueue_scan_indexing` from Phase 6 Plan 06-06). Worker just persists whatever the CLI emitted in `summary` key (if present), letting the list page render counts pre-indexing.
- **Local stub_stripe_meter and _wire_db_to_pg fixtures inlined.** Plan 06 is the first taskiq-job-with-DB+Stripe in the `jobs/` namespace. Promoting to `backend/tests/jobs/conftest.py` would over-design before a second consumer exists.
- **Two timeout error wrap.** `asyncio.TimeoutError` (post-`UP041`: just `TimeoutError`) is re-raised as `RuntimeError(f"... timed out after {N}s") from toexc`. The wrapped RuntimeError carries a precise scan_repo-specific message that the failure-path UPDATE writes verbatim into `scans.error_message` for the polling UI to render.
