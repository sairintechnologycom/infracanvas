# Phase 1: Canvas MVP - Context

**Gathered:** 2026-04-16
**Status:** Ready for planning

<domain>
## Phase Boundary

Engineers install InfraCanvas once (`pip install infracanvas` or `brew install infracanvas`) and run one command against any Terraform directory. They get an interactive, annotated infrastructure diagram with security findings and a score card in under 10 seconds. The tool opens their browser automatically. This phase delivers the complete CLI-first product — parser, graph, security engine, viewer, score card, and distribution.

This phase does NOT include: Azure support, live cloud API scanning, SaaS dashboard, team collaboration, or watch-mode file serving beyond basic serve command.

</domain>

<decisions>
## Implementation Decisions

### Free Tier Gate (VWR-06)

- **D-01:** Gate strategy — Show: finding count per node, severity badges (Critical/High/Medium), overall score letter grade. Blur: finding title, finding description, remediation guidance, resource attribute evidence. Users see there's a problem and how many, but not what it is or how to fix it.
- **D-02:** Upgrade CTA links to `infracanvas.dev` founding member Stripe page ($49/mo locked pricing). Every scan is a conversion opportunity — consistent with the Phase 0 funnel.
- **D-03:** Free mode is hardcoded in the exported HTML for Phase 1. No license key, no unlock mechanism. Paid/unlocked mode is deferred to Phase 4 SaaS dashboard. Keep Phase 1 simple.

### 15 AWS Resource Types (PRS-02)

- **D-04:** Core 6 (definite): `aws_vpc`, `aws_subnet`, `aws_security_group`, `aws_instance`, `aws_s3_bucket`, `aws_iam_role`
- **D-05:** Remaining 9 slots filled from managed services — focus on what modern Terraform repos actually contain: `aws_db_instance` (RDS), `aws_lambda_function`, `aws_lb`/`aws_alb` (ALB/NLB), `aws_eks_cluster`. Exact final 15 is Claude's discretion — prioritise resources where security rules add the most value (finding-rich types).
- **D-06:** Unrecognised resource types (anything not in the supported 15) render as a generic node with resource type label — visible in the diagram, no icon, no security rules applied. No silent drops.

### Score Card Design (SCR-01 through SCR-03)

- **D-07:** `infracanvas score` outputs a standalone `infracanvas-score.html` file (separate from the main diagram HTML). Opens in browser.
- **D-08:** Score card layout: large letter grade (A/B/C/D/F) + numeric score (X/100) at top → 5 progress bar rows (Security, Encryption, IAM Hygiene, Cost Efficiency, Tagging each with dimension score) → footer with resource count + finding count + [Upgrade ↗] CTA linking to infracanvas.dev.
- **D-09:** Small "Generated by InfraCanvas · infracanvas.dev" footer attribution on every score card. Viral loop — every shared screenshot or Slack post is a referral.

### CLI UX + Browser Open (EXP-02, CLI-01)

- **D-10:** `infracanvas scan ./terraform` → saves `infracanvas-report.html` in current directory AND auto-opens it in the default browser. This is the core "magical" UX — the success criteria explicitly calls for browser opening.
- **D-11:** CI auto-detection: skip browser open when `DISPLAY` env var is unset (Linux headless) OR any of `CI=true`, `GITHUB_ACTIONS`, `CIRCLECI`, `TRAVIS`, `JENKINS_URL` are set. Print file path instead: `Report saved: ./infracanvas-report.html`. Zero config required from the user.
- **D-12:** `infracanvas serve ./terraform` → local HTTP server + file watcher. Re-scans `.tf` files on change, browser auto-refreshes. Basic watch-mode experience. Delivers the `serve` command from CLI-01 skeleton with real utility rather than being an alias.

### Claude's Discretion

- Exact final selection of 15 AWS resource types (within the managed services category guidance above)
- Score card color scheme and visual polish
- Progress bar style and animation details
- File naming convention for exported artifacts
- Terminal output formatting (Rich tables, progress indicators)
- HTTP port for `serve` command (default 8080 or similar)
- Debounce timing for file watcher in `serve` mode

</decisions>

## Existing Code Insights

### Reusable Assets
- `viewer/src/components/DiagramCanvas.tsx` — Full ReactFlow diagram; needs free-gate overlay added to finding detail
- `viewer/src/components/DetailPanel.tsx` — Finding detail panel; this is where blur/gate UI gets added
- `viewer/src/components/SummaryBar.tsx` — Top bar with score and counts; already shows the data needed for score card
- `viewer/src/lib/colors.ts` — Severity color mapping (reuse for score card dimension bars)
- `cli/infracanvas/security/scorer.py` — Score calculation; output feeds into both SummaryBar and the standalone score card HTML
- `cli/infracanvas/export/scorecard.py` — Score card renderer (exists; likely needs redesign to match D-08 layout)
- `cli/infracanvas/security/rules/aws/` — Existing YAML rules (s3, database, compute, networking, iam)
- `cli/tests/fixtures/` — 8 test fixture scenarios (clean_infra, insecure_setup, multi_module, etc.)

### Established Patterns
- Single-file HTML export via `vite-plugin-singlefile` — both diagram and score card should follow this pattern
- Data injection via `window.__INFRACANVAS_DATA__` — score card can use same mechanism
- Zustand store with `graph`, `selectedNode`, `filters` — adding a `gateMode` boolean is the natural extension for free-gate
- YAML-driven security rules — adding new rules requires no code changes, just a new `.yaml` file

### Integration Points
- `cli/infracanvas/export/html.py` → injects `ResourceGraph` JSON into viewer template
- `cli/infracanvas/main.py` → `scan` command orchestrates the full pipeline; browser open logic goes here
- `viewer/src/App.tsx` → reads `window.__INFRACANVAS_DATA__`; gate mode flag can be injected alongside graph data
- `cli/infracanvas/security/scorer.py` → produces score dimensions; standalone score card HTML reads these

</code_context>

<specifics>
## Specific Ideas

- Score card mockup confirmed by user: letter grade prominent at top, 5 horizontal progress bars with dimension scores, resource/finding count in footer, [Upgrade ↗] CTA, "Generated by InfraCanvas · infracanvas.dev" attribution
- The free-gate blur should feel intentional — not broken. Show the count ("3 Critical findings") and a lock icon, with the upgrade CTA nearby. Not just CSS blur with no explanation.
- `infracanvas serve` should feel like `hugo serve` or `vite dev` — a development-loop-friendly command engineers can leave running while they work on their Terraform
