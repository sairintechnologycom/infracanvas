## Spacing Scale

| Token | Value | Usage |

## Typography

All text renders on dark backgrounds. Use `font-sans` (Inter) for all UI chrome and labels. Use `font-mono` (JetBrains Mono) for resource IDs, Terraform resource type strings, attribute values, evidence JSON, and code blocks.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 12px (text-xs) | 400 regular | 1.5 | sans | Finding title, panel labels, tab labels, general readable content |
| Label | 10px (text-[10px]) | 400 regular | 1.4 | sans | Severity badges, resource type chips, uppercase section headers (tracking-wider), finding count pills, score card dimension labels |
| Mono-body | 11px (text-[11px]) | 400 regular | 1.5 | mono | Resource IDs, attribute values, finding rule IDs, dependency list items |
| Heading | 14px (text-sm) | 600 semibold | 1.2 | sans | Panel resource name, score card letter grade sub-label, SummaryBar project name |
| Display | 72px | 600 semibold | 1.0 | sans | Score card letter grade (A/B/C/D/F) — single character, centred, coloured by grade |

Note: Existing codebase uses `text-sm` (14px) for semibold names and `text-[11px]`/`text-[10px]` for dense data labels. This 4-level scale (`10 / 11 / 12 / 14`) is consistent throughout — do not introduce 16px or larger in the viewer or score card HTML. The score card letter grade at 72px is a deliberate display exception for the standalone `infracanvas-score.html` only. Only two font weights are permitted: 400 (regular) for body, label, and mono roles; 600 (semibold) for heading, badge, CTA, and display roles.

## Free-Gate UI Contract (VWR-06)

This is the primary new interaction in Phase 1. The gate must feel intentional and informative — not broken.

### Gate trigger
`window.__INFRACANVAS_GATE__ === true` (injected by `export_html()` in all Phase 1 HTML exports).

### FindingsTab in gate mode
When `gateMode === true` AND `node.findings.length > 0`, replace the `FindingCard` list with:

```
[Lock icon 16px, color: #64748b]

[N finding(s)]  [Critical badge] [High badge] [Medium badge]
                                                (severity badge chips, colour-coded)

[Blurred placeholder block — 3 rows of fake finding text, CSS blur(4px)]
  background: #111827, border: 1px solid #1e293b

[Unlock full findings]  →  links to https://infracanvas.dev/founding
CTA button: background #3b82f640, border 1px solid #3b82f6, color #60a5fa
text: "Unlock details — $49/mo founding member"
font-size: 11px, font-weight: 600, border-radius: 6px, padding: 8px 16px
```

The finding count and severity badges are always fully visible (never blurred). Only the finding title, description, evidence, and remediation are obscured. The blur placeholder rows must be 3 distinct block elements with `filter: blur(4px)` and `pointer-events: none` — not a single tall block.

### FindingCard in gate mode
`FindingCard` receives a `gateMode: boolean` prop. When `true`:
- Show: severity icon, severity badge, rule_id chip
- Hide (replace with blurred placeholder text): `finding.title`, `finding.description`, `finding.evidence`, `finding.remediation`
- Blurred copy reads: "████ ███ ██████ ████" (use `opacity: 0` text + `filter: blur(4px)` background block approach, NOT actual characters)

### ResourceNode badge
In gate mode, the finding count badge (top-right of node) remains fully visible — this is explicitly part of D-01. No changes to ResourceNode badge behaviour.

### OverviewTab findings summary
The severity chip row in `OverviewTab` ("Findings Summary" section) remains fully visible in gate mode. Only the Findings tab content is gated.

## Interaction Contracts

### Node click
- Click ResourceNode → `setSelectedNode(node)` → DetailPanel slides in from right
- DetailPanel default tab: `overview` (unchanged)
- Clicking same node again: no-op (already selected)
- Clicking canvas background: `setSelectedNode(null)` → DetailPanel closes

### Gate overlay click
- Clicking anywhere on the blurred finding rows: no action (pointer-events: none on blur layer)
- Clicking "Unlock details" CTA: opens `https://infracanvas.dev/founding` in new tab (`target="_blank" rel="noopener noreferrer"`)
- Lock icon: decorative, no interaction

### Search (VWR-05)
- Input location: SummaryBar right section, before Filters button
- Behaviour: filter nodes by `data.name.includes(query) || data.type.includes(query)`
- Matching: case-insensitive
- Non-matching nodes: opacity 0.2 (same pattern as severity filter)
- Empty query: all nodes at full opacity
- No debounce needed for Phase 1 (simple string match, no network)

### Zoom / minimap (VWR-05)
- Controls: existing ReactFlow Controls component, styled per `index.css` overrides (already present)
- Minimap: existing ReactFlow MiniMap, styled per `index.css` overrides (already present)
- No changes needed

### Score card (standalone HTML, no interactivity)
- Static render, no click targets except "Upgrade" CTA button
- "Upgrade" CTA: links to `https://infracanvas.dev/founding`, `target="_blank" rel="noopener noreferrer"`
- "infracanvas.dev" attribution footer: links to `https://infracanvas.dev`, same target
