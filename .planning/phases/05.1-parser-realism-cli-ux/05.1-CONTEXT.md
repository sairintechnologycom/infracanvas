# Phase 5.1: Parser Realism + CLI UX — Context

**Gathered:** 2026-04-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Harden the existing Terraform parser + `infracanvas` CLI so that a realistic production codebase (root module calling local submodules with `var`/`count`/`data`) produces an accurate diagram and a clean one-command "open the HTML" user flow.

**What this phase delivers:**
- Submodule parse errors surface to the user instead of being silently swallowed
- `count = <literal>` expands into multiple graph nodes (one per instance)
- CLI gains `--quiet` and `--open` flags for the "just render the picture" UX
- A realistic envs-layout test fixture that exercises the above end-to-end

**What this phase does NOT touch:**
- Registry module sources (`terraform-aws-modules/...`) — deferred to v1.2
- Full `var.*` / `local.*` / `data.*` resolution across module boundaries — deferred
- `for_each` with unresolved maps — fallback to collapsed+badge like unresolved `count`
- SaaS backend / dashboard / webhooks — Phase 6+

We'll clarify HOW to implement what's in scope. New capabilities belong in other phases.
</domain>

<investigation>
## Pre-planning Investigation

Manual scan of a handwritten realistic fixture (`/tmp/ic-real/envs/prod` calling `../../modules/vpc` with `count = var.az_count`) surfaced the current state:

**Works today:**
- `cli/infracanvas/parser/module.py::resolve_modules` correctly follows `./` and `../` sources, enforces depth limit 3, detects circular refs, and prefixes submodule resources with `module.{name}`
- `main.py:98` wires the resolver into the scan pipeline

**Gaps surfaced:**
- **Silent parse swallow:** invalid HCL in a submodule results in 0 resources added and no stderr output. User sees an incomplete diagram with no signal that something is broken.
- **No `count` expansion:** `aws_subnet.public` with `count = 3` renders as 1 node, not 3.
- **Unresolved `var.*` in attributes:** a resolved submodule's resources keep literal `${var.cidr}` in attribute values instead of the caller's passed-in literal.
- **No fixture:** no existing test under `cli/tests/fixtures/` uses a `module {}` block, so regressions in module resolution are uncatchable.
- **CLI noise:** every `scan` dumps the full Rich findings table to stdout. No `--quiet` or `--open` flag.

</investigation>

<decisions>
## Implementation Decisions

### Submodule Parse Errors
- **D-01: Warn + continue.** When a submodule fails to parse, emit a clear stderr warning per broken file (with path + parser error snippet), render a placeholder node in the diagram marking the unresolved module, and exit 0. Rationale: matches Terraform's best-effort philosophy; partial diagrams remain useful; doesn't turn a single `.tf` typo into a CI gate failure.

### `count` / `for_each` Expansion
- **D-02: Expand when literal.** If `count = 3` (or any literal integer), expand into N graph instances with IDs `module.vpc.aws_subnet.public[0..N-1]`. If `count = var.az_count` or similar unresolved expression, keep collapsed as a single node with a `×?` badge. `for_each` on a literal map follows the same pattern; unresolved `for_each` also collapses+badges. Full `var.*`/`local.*` resolution deferred to a later phase.

### CLI UX — `--quiet` + `--open`
- **D-03: One-line summary on success.** With `--quiet`, successful runs print exactly one line to stdout: `✓ {N} resources · {M} findings · score {S} · opened in browser` (omit the `· opened in browser` tail when `--open` not set). Errors continue going to stderr with full detail. Without `--quiet`, existing Rich table output is preserved for backward compatibility. `--open` triggers `open`/`xdg-open`/`start` based on platform, only when the output format is HTML — error out if `--open` is combined with a non-HTML format.

### Test Fixture
- **D-04: Envs-layout fixture.** New fixture at `cli/tests/fixtures/envs_layout/` with the shape:
  ```
  envs_layout/
    envs/prod/main.tf     (calls ../../modules/vpc with count=3, uses data.aws_caller_identity)
    modules/vpc/main.tf   (declares variables, uses count, outputs)
  ```
  Single fixture covers end-to-end module resolution + count expansion + variable passthrough. Parser unit tests assert resource count, ID format, and graceful degradation on a deliberately broken sibling submodule.

### Claude's Discretion
- Exact wording of stderr warnings (e.g., `⚠ Module 'vpc' at modules/vpc: parse error on line 3`).
- Visual styling of the placeholder node in the viewer (can follow existing drift "shadow" pattern).
- Whether literal detection uses AST inspection or a simple isinstance(int) check on the parsed value — up to implementer.
- One-line summary format details (exact emoji/symbol choice, color when TTY).
- Cross-platform `--open` implementation (Python's `webbrowser` module is the obvious choice).
- Exit code semantics for `--open` when browser fails to launch (recommend: log to stderr, don't change exit code).

</decisions>

<specifics>
## Specific Ideas

- **Partial-scan UX:** inspired by how `terraform plan` continues despite warnings but surfaces them prominently. Don't copy verbatim; we have more UI canvas to use (the placeholder node itself is the signal).
- **One-line summary symbol:** `✓` on success, consistent with Rich defaults. Gate `✓` on TTY detection — pipe-safe fallback should be ASCII-clean.
- **Envs-layout fixture naming:** use `envs_layout` (not `envs` — less ambiguous) and document its purpose in a short README inside the fixture dir so future maintainers know what it's stress-testing.

</specifics>

<deferred>
## Deferred Ideas

Captured during Phase 5.1 manual investigation but deliberately out of scope:

- **Registry module resolution** (`source = "terraform-aws-modules/vpc/aws"`) — requires `terraform init` equivalent, network fetch, version pinning. Deferred to v1.2 Enterprise.
- **Full cross-module variable resolution** — following `var.*`, `local.*`, `data.*` from caller scope so submodule resources show concrete values instead of interpolation strings. Deferred — literal substitution in this phase is a meaningful stepping stone.
- **`terraform_remote_state` data source** — treat as opaque for now.
- **`.tfvars` file loading** — unclear UX (which file? CLI flag? auto-detect?); defer until there's a user asking.
- **`for_each` over unresolved maps** — collapses+badges this phase; full expansion deferred.
- **`serve` watcher missing atomic-replace writes** — surfaced during manual testing; separate bug, not in this phase's scope. File under known limitations or spin off a separate quick-fix phase.

</deferred>
