## Pre-discussion Investigation

Before presenting gray areas, a manual scan of a realistic handwritten fixture confirmed the actual state of the parser — the `resolve_modules` function is already wired in and works on valid HCL. Invalid HCL in a submodule was silently swallowed, which reshaped the gray areas from "build module resolution" to "harden what exists."

---

## Submodule Parse Errors

| Option | Description | Selected |
|--------|-------------|----------|
| Warn + continue | Stderr warning per broken submodule, render placeholder node, exit 0. Matches Terraform's best-effort UX; keeps partial scans useful. | ✓ |
| Fail loud | Exit non-zero with clear error listing broken files. Safer for CI gates but blocks diagram on any syntax slip. | |
| Keep silent, add --verbose | Current behavior as default; errors only with --verbose. Least disruption but keeps the silent-swallow footgun. | |

**User's choice:** Warn + continue (Recommended).
**Notes:** Chosen for the partial-scan usefulness and Terraform-aligned UX.

---

## count / for_each Expansion

| Option | Description | Selected |
|--------|-------------|----------|
| Expand when literal | Literal int count → N nodes. Unresolved (`count = var.x`) → collapsed with `×?` badge. Covers most real cases. | ✓ |
| Full expansion incl. vars | Resolve `var.*`/`local.*` from caller, then expand. Most faithful to `terraform plan` but doubles phase scope. | |
| Defer — out of scope | Document as known limitation, carry into a future phase. Keeps 5.1 small but accepts less-realistic diagrams. | |

**User's choice:** Expand when literal (Recommended).
**Notes:** Pragmatic middle ground — gives real-Terraform fidelity for the common case without expanding scope into full variable resolution.

## CLI --quiet / --open Contract

| Option | Description | Selected |
|--------|-------------|----------|
| One-line summary | `✓ 42 resources · 7 findings · score 62 · opened in browser`. Keeps signal on golden path, drops the Rich table. Errors still stderr. | ✓ |
| Silent on success | No stdout on success. Unix-canonical but less informative. | |
| --open only, keep existing output | Skip --quiet this phase; just add --open. Minimizes change but leaves noisy-CLI unaddressed. | |

**User's choice:** One-line summary (Recommended).
**Notes:** Biases toward human readability while still being pipe-safe and quiet relative to the current default.

## Test Fixture Shape

| Option | Description | Selected |
|--------|-------------|----------|
| Envs layout | `envs/prod/main.tf` calls `../../modules/vpc` with vars + count. Matches production Terraform layout. | ✓ |
| Flat root + ./submodule | Single `main.tf` calls `./modules/vpc`. Simpler but less representative of real projects. | |
| Both | Envs-style for end-to-end + flat for unit tests. More coverage, more maintenance. | |

**User's choice:** Envs layout (Recommended).
**Notes:** Prioritizes realistic demo credibility over unit-test granularity. A deliberately broken sibling submodule in the same fixture covers the parse-error path without needing a second fixture dir.
