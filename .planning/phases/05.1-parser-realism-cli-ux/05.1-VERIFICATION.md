## Known Issues (Pre-Existing, Documented)

### WR-01: Edge source IDs miss module prefix and `[index]` suffix

**File:** `cli/infracanvas/graph/builder.py:85`
**Severity:** warning (pre-existing bug newly exposed)
**Origin:** Pre-dates Phase 5.1; surfaced by Plan 02's module-prefixing and bracket-suffix logic.

**Issue:** `_build_edges` computes `source_id = f"{res.resource_type}.{res.name}"` — no module prefix, no `[index]` suffix. But `resource_ids = {n.id for n in nodes}` contains fully-qualified IDs (`module.vpc.aws_subnet.public[0]`). The `if dep in resource_ids` check therefore drops every edge originating from a module-scoped or count-expanded resource. Additionally, because `_build_edges` iterates `parsed.resources` (not `nodes`), count-expanded resources contribute N identical loop iterations and emit N-duplicate implicit edges.

**Impact on SC-01:** SC-01 requires "resources appear in the graph with module prefix" — this is about NODE IDs, which ARE correctly prefixed (verified by test 5.1-A). Edge connectivity for module-scoped resources is a separate correctness concern not explicitly listed in SC-01.

**Disposition:** Does NOT block phase completion. Recommend filing as a standalone plan in a subsequent edge-correctness phase. Per 05.1-REVIEW.md, the fix requires rewriting `implicit_deps` to carry module-aware targets OR resolving target IDs at edge-build time by scanning `nodes` for name/type matches within the same module.
