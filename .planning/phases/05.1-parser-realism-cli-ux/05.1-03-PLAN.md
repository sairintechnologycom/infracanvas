<semantic_collision_decision>
**DECIDED (per orchestrator critical_planning_decision + CONTEXT.md D-03):** Repurpose `--quiet` to the D-03 semantics ("one-line summary on success"). Introduce a NEW `--json` flag to carry the OLD `--quiet` behavior (JSON to stdout, exit 0 regardless of findings).

Why not option (b) of routing through `--ci`:
- `--ci` currently couples JSON-dump WITH exit-code-on-findings (main.py:358–366). Tests at test_cli.py:17, :31, :82, :112 used `--quiet` to get JSON-to-stdout WITHOUT the exit-code coupling. Merging them breaks `test_scan_simple_vpc` (simple_vpc has findings → `--ci` would exit 1, test asserts 0).
- A new `--json` flag is the cleanest split: three orthogonal flags for three output shapes —
  - `--quiet` = one-line summary (D-03)
  - `--json`  = JSON to stdout, exit 0 on success (OLD `--quiet` behavior)
  - `--ci`    = JSON to stdout + exit 1/0 gated by `--fail-on`/`--severity` (unchanged)

DO NOT shell out to `open`/`xdg-open`/`start` directly. `webbrowser` opens via platform APIs without invoking a shell; `file://` URIs are not expanded. `out_path.resolve().as_uri()` produces a valid RFC-8089 URI with no shell metacharacters. |
