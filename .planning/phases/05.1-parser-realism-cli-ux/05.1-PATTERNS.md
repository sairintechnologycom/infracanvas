### `cli/infracanvas/parser/hcl.py` — count-expansion hook

**Analog:** self. The cleanest expansion site is inside `_extract_resources`, right where each parsed `instance` dict is converted into a `ParsedResource`.

**Current pattern** (lines 105–125):
```python
def _extract_resources(parsed: dict[str, Any], result: ParsedTerraform) -> None:
    """Extract resource blocks from parsed HCL."""
    for resource_block in parsed.get("resource", []):
        for resource_type_raw, instances in resource_block.items():
            resource_type = _strip_quotes(resource_type_raw)
            items = instances if isinstance(instances, list) else [instances]
            for instance in items:
                if isinstance(instance, dict):
                    for name_raw, attrs in instance.items():
                        name = _strip_quotes(name_raw)
                        attrs_dict = _clean_value(attrs) if isinstance(attrs, dict) else {}
                        depends_on_raw = attrs_dict.pop("depends_on", [])
                        depends_on = _normalize_depends_on(depends_on_raw)
                        result.resources.append(
                            ParsedResource(
                                resource_type=resource_type,
                                name=name,
                                attributes=attrs_dict,
                                depends_on=depends_on,
                            )
                        )
```

**Extension strategy (D-02):** inside the inner loop, after `attrs_dict` is built, inspect `attrs_dict.get("count")`:
- If it is an `int` (or castable via `isinstance(val, int)` check on the `hcl2`-parsed literal) → emit N `ParsedResource` objects. Naming convention must match Terraform addressing: keep `name` unchanged, but attach an `index` (new field on `ParsedResource`) so the graph builder can produce IDs like `aws_subnet.public[0]`.
- If it is anything else (list wrapping a `${var.foo}` string, dict, etc.) → emit a single `ParsedResource` and flag `unresolved_count=True` (new field) so the graph builder can render the `×?` badge.

Apply same `for_each` inspection with identical fallback (literal map → expand; anything else → collapsed+badged).

**Field additions to `ParsedResource`** (around line 14):
```python
@dataclass
class ParsedResource:
    resource_type: str
    name: str
    attributes: dict[str, Any]
    depends_on: list[str] = field(default_factory=list)
    module: str = ""
    index: int | None = None              # NEW — set when expanded from count/for_each
    unresolved_count: bool = False        # NEW — set when count/for_each is non-literal
```

**Helper placement:** add `_expand_count(attrs_dict) -> list[tuple[int | None, dict]]` and `_expand_for_each(...)` as module-level `_`-prefixed helpers, consistent with existing `_strip_quotes`, `_clean_value`, `_normalize_depends_on`, `_strip_interpolation` on lines 42–223.

### `cli/infracanvas/parser/module.py` — parse-error surfacing

**Analog:** self. `resolve_modules` already calls `parse_directory`, which populates `sub_parsed.parse_errors` — the list is simply never merged back into the caller's `parsed.parse_errors`.

**Current pattern** (lines 34–57):
```python
for module_block in parsed._raw_modules:
    source = module_block.get("source", "")
    name = module_block.get("__name__", "unknown")

    if not (source.startswith("./") or source.startswith("../")):
        continue

    module_dir = (directory / source).resolve()

    if not module_dir.is_dir():
        continue

    if module_dir in _visited:
        continue

    sub_parsed = parse_directory(module_dir)
    for res in sub_parsed.resources:
        res.module = f"module.{name}"
    parsed.resources.extend(sub_parsed.resources)

    resolve_modules(module_dir, sub_parsed, depth + 1, _visited)
```

**Extension strategy (D-01):**
1. After `sub_parsed = parse_directory(module_dir)`, merge its errors up: `parsed.parse_errors.extend(sub_parsed.parse_errors)`.
2. If `sub_parsed.resources` is empty AND `sub_parsed.parse_errors` is non-empty, synthesize a placeholder `ParsedResource` with a synthetic type (e.g., `_infracanvas_unresolved_module`) so the graph builder renders it. Use the caller-passed module `name` and attach evidence (`source`, first parse error) via the `attributes` dict — consistent with how other placeholder/shadow resources propagate metadata into `ResourceNode.attributes`.
3. The separate case where `source` is a registry ref (`terraform-aws-modules/vpc/aws`) or `module_dir` is not a directory is explicitly deferred (see `<deferred>` in CONTEXT.md). For now those paths silently `continue` as they do today.

**Do not rewrite** `resolve_modules` — it already handles depth limit (line 25), circular detection (line 49), and module-prefix annotation (line 54). Only add the three surfacing hooks above.

### `cli/infracanvas/main.py` — `--quiet` / `--open` wiring

**Analog:** self. The `scan` command already has every plumbing piece we need.

**Existing `--quiet` conflict:** `scan` already exposes `--quiet` (line 290–293), but its current semantics are "JSON only to stdout, for CI/CD" — i.e. it overlaps with `--ci`. **CONTEXT.md D-03 redefines `--quiet` to mean "one-line summary on success."** The plan must either (a) repurpose the flag and adjust the single existing call site in `test_cli.py:17` that relies on JSON-to-stdout behavior, or (b) route the current JSON-to-stdout behavior entirely through `--ci` (which already emits JSON to stdout, line 354–366) and assign `--quiet` the new one-line-summary role. Option (b) is the cleaner split — the planner should call this out explicitly.

**Existing browser-open plumbing** (lines 61–68, reuse unchanged):
```python
def _should_open_browser() -> bool:
    """Return False when running in CI/headless environment (per D-11)."""
    ci_env_vars = ["CI", "GITHUB_ACTIONS", "CIRCLECI", "TRAVIS", "JENKINS_URL"]
    if any(os.environ.get(v) for v in ci_env_vars):
        return False
    if not os.environ.get("DISPLAY"):
        return False
    return True
```

```python
# Line 376–377 (inside scan, html branch)
if config.open_browser and _should_open_browser():
    webbrowser.open(out_path.resolve().as_uri())
```

**Extension strategy (D-03):**
1. Add `--open` flag to `scan` (and optionally `score`, `plan`) using the same `Annotated[bool, typer.Option(...)]` idiom as every other flag on lines 290–334. Default `False`.
2. When `--open` is combined with `--format json` or `--format=non-html`, error out at the top of `scan` with `_err_console.print(...)` + `raise typer.Exit(code=2)` — mirrors the existing `directory.is_dir()` guard on line 337.
3. When `--open` is set and format is html, call `webbrowser.open(out_path.resolve().as_uri())` unconditionally (bypass `_should_open_browser()`'s CI detection — the user explicitly asked for it). If `webbrowser.open` returns `False`, print a stderr warning but do not change exit code (per D-03 discretion note).
4. One-line summary format (when `--quiet` is set and run succeeds):
   ```python
   # After graph is built, before the Rich table path:
   summary_line = (
       f"✓ {graph.summary.total_resources} resources · "
       f"{sum(graph.summary.findings.values())} findings · "
       f"score {graph.summary.score}"
   )
   if open_flag:
       summary_line += " · opened in browser"
   typer.echo(summary_line)
   ```
   Gate `✓` on `sys.stdout.isatty()` for pipe-safe fallback (use `"OK"` when not a TTY) — matches the `<specifics>` note in CONTEXT.md.

**Parse-error stderr pattern** (D-01 tail):
The existing loop at lines 92–94 already prints `parsed.parse_errors`:
```python
if parsed.parse_errors:
    for path, err in parsed.parse_errors:
        out.print(f"[yellow]Warning:[/yellow] Could not parse {path.name}: {err}")
```
After the `module.py` change merges submodule errors into `parsed.parse_errors`, this existing loop automatically covers D-01's stderr warning requirement — **no new code needed here**, just route `out` to `_err_console` (currently goes to `console` or `_ci_console`). Planner must verify exit code stays 0 per D-01.
