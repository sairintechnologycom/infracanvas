<objective>
Create Alembic migration 009 that adds `slack_webhook_url TEXT NULL` to the `teams` table and creates the `team_id_for_installation(bigint) RETURNS uuid SECURITY DEFINER` SQL function. Also update the Team SQLAlchemy ORM class to reflect the new column. The SECURITY DEFINER function is required because the GitHub webhook handler has no RLS context (no Clerk JWT) — it cannot call `set_config('app.current_team_id', ...)` before it knows the team_id. The function bypasses RLS to do a direct lookup from `github_installations` and return the owning team's UUID.
</objective>

<threat_model>

| Threat | Mitigation |
|--------|-----------|
| T-8-01-A: SECURITY DEFINER function used to exfiltrate cross-team data | Function only SELECTs `team_id FROM github_installations WHERE github_installation_id = $1` — single column, single equality predicate, no joins to other teams' data |
| T-8-01-B: Slack webhook URL stored in DB and leaked via API | Column is never returned in scan list/get responses; only read internally by the worker |

</threat_model>
