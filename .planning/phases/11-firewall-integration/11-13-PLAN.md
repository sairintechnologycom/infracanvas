## Phase 11 — Firewall Integration

Phase 11 extends the agent with 4 firewall vendors via 5 protocols, fed by a 4th ticker on 1h cadence (D-02). Per CONTEXT decisions, no new schema is added beyond extending the existing `devices[]` array with new protocol values.

### Protocols and Collectors

| Protocol Value | Collector | Auth | Reference |
|---|---|---|---|
| asa-rest | agent/internal/asa.RESTCollector | POST /api/tokenservices → X-Auth-Token | ASA-01 |
| asa-ssh | agent/internal/asa.SSHCollector | SSH password (terminal pager 0; show running-config) | ASA-03 |
| fmc | agent/internal/fmc.Client | POST /api/fmc_platform/v1/auth/generatetoken → X-auth-access-token | ASA-02 |
| checkpoint | agent/internal/checkpoint.LiveCollector | POST /web_api/login → X-chkp-sid; logout per-pull | CKP-01 |
| checkpoint-import | agent/internal/checkpoint.LoadImport | (file read, no network) | CKP-02 |

### Data Flow

1. The 4th ticker fires every 1h (Intervals.Firewall, agent/cmd/infracanvas-agent/main.go).
2. For each device whose protocol is one of the 5 firewall protocols, the dispatcher mints a UUIDv4 `snapshot_id` (Pattern 2).
3. The vendor collector pulls rules + NAT + objects.
4. The agent fans out three POSTs to the backend, all sharing the same `snapshot_id`:
   - POST /v1/agent/firewall-rules
   - POST /v1/agent/firewall-nat
   - POST /v1/agent/firewall-objects
5. The backend uses INSERT ... ON CONFLICT DO NOTHING on the parent `firewall_ruleset_snapshots` row, so the three handlers are independent and idempotent regardless of arrival order or repetition.

### Storage Model

Hybrid normalized + raw_blob (D-08): each firewall_rules row has both vendor-agnostic columns (src_cidr, dst_cidr, action, protocol, ports, etc.) and a JSONB `raw_blob` preserving the vendor-native rule. Phase 12 path computation reads the normalized columns; vendor-specific UI/audit reads `raw_blob`.
```

## Phase 11 Firewall Integration

| ID | Limitation | Impact | Mitigation / Workaround |
|----|------------|--------|------------------------|
| L-11-01 | Cisco ASA REST API removed at ASA 9.17+ | asa-rest protocol fails with 404 on devices running 9.17+ | Operator switches device to protocol: asa-ssh (RESEARCH Pitfall 1) |

| L-11-04 | FMC integration uses first DOMAIN_UUID + first access policy + first NAT policy returned | Multi-tenant FMC or multi-policy deployments only get the first policy | Operator-driven policy selection deferred to a future phase (RESEARCH Pitfall 6) |
| L-11-05 | Snapshot retention default 14 days | Older snapshot history not queryable | Adjustable via FIREWALL_SNAPSHOT_TTL_DAYS env var on backend (RESEARCH Risk Landmine 2) |

### Generating checkpoint-import Files

On the Checkpoint mgmt host:
```bash
mgmt_cli login user "$U" password "$P" > session.txt
mgmt_cli show access-rulebase name "Standard" --format json -s session.txt > cp-airgap.rulebase.json
mgmt_cli show nat-rulebase package "Standard" --format json -s session.txt > cp-airgap.nat.json
mgmt_cli show objects type any --format json -s session.txt > cp-airgap.objects.json
mgmt_cli logout -s session.txt
```
Copy the three files to the agent host's `/etc/infracanvas/` directory with `chown infracanvas:infracanvas` and `chmod 600`.

### Credential Rotation

1. Rotate password in firewall management UI / mgmt_cli
2. Update `password:` field for the device in `agent.yaml`
3. `systemctl reload infracanvas-agent` (or send SIGHUP if managed without systemd)
4. Watch logs for next firewall_pull tick — should succeed within 1h. If 401 errors persist after the next tick, verify password and account status.

### Troubleshooting

| Symptom | Likely Cause | Action |
|---------|--------------|--------|
| 401 from asa-rest endpoint | ASA 9.17+ removed REST API | Switch device to protocol: asa-ssh (L-11-01) |
| 401 from FMC after a few hours | Token TTL exceeded; refresh failed | Check FMC user account is not locked; agent automatically re-acquires on next tick |
| SID timeout / pagination cut off mid-pull on Checkpoint | Rule layer exceeds session timeout | Already passing session-timeout=3600 at login; for layers >50k rules use checkpoint-import (L-11-02) |
| checkpoint-import: file not found | Sibling file naming mismatch | Confirm all three files exist with .rulebase.json / .nat.json / .objects.json suffixes (L-11-03) |
| asa-ssh parser misses rules | ASA running-config format differs from regex | Submit a sample anonymized config to support; parser regex may need extension (RESEARCH Risk Landmine 5) |
