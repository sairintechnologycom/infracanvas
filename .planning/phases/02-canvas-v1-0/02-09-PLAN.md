## Gap

Exported `infracanvas.html` fetches `https://fonts.googleapis.com/css2?family=JetBrains+Mono...&family=Inter...&display=swap` at load time, violating Phase 1 success criterion "opens in any browser with zero dependencies."

## Root Cause

`viewer/index.html:7-9`:

```html
<style>
  @import url('https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@400;500;600&family=Inter:wght@400;500;600;700&display=swap');
</style>
```

`vite-plugin-singlefile` inlines local script/CSS assets but does **not** rewrite external `@import url(https://...)` — that remains a runtime network fetch. No `@fontsource/*` dependency is installed (`viewer/package.json` checked).

## Fix — Inline via @fontsource (Option 1 from UAT fix_hint)

Keeps the JetBrains Mono + Inter aesthetic. Bundles woff2 files as base64 inside the single HTML. Zero runtime fetches.

## Tasks

### T-01 — Add font dependencies

```bash
cd viewer
npm install --save @fontsource/inter @fontsource/jetbrains-mono
```

Expected: `package.json` gains two `@fontsource/*` deps; `package-lock.json` updates.

### T-02 — Import fonts in the entry module

Edit `viewer/src/main.tsx`, add near the top of the imports:

```ts
import '@fontsource/inter/400.css'
import '@fontsource/inter/500.css'
import '@fontsource/inter/600.css'
import '@fontsource/inter/700.css'
import '@fontsource/jetbrains-mono/400.css'
import '@fontsource/jetbrains-mono/500.css'
import '@fontsource/jetbrains-mono/600.css'
```

These CSS files `@font-face`-declare local woff2 URLs; Vite resolves them through the module graph so `vite-plugin-singlefile` can inline them.

### T-03 — Remove the Google Fonts @import

Edit `viewer/index.html`: delete lines 7–9 (the `<style>@import url(...)</style>` block).
Leave the `<script>window.__INFRACANVAS_DATA__ = null;</script>` line intact.

### T-04 — Verify assetsInlineLimit covers woff2

`viewer/vite.config.ts:10` already sets `assetsInlineLimit: 100000000` (100MB), so woff2 files (~30–50KB each) are inlined automatically. No config change needed. Confirm by re-reading the file after T-03.
