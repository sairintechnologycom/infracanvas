## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, inline gaps (gap-1, p-1) |
| sm | 8px | Compact element spacing, tab inner padding (px-2 py-2, gap-2) |
| md | 16px | Panel section padding, card internal padding (p-4) |
| lg | 24px | Panel section gaps, group container padding |
| xl | 32px | Drawer/panel width steps, major layout gaps |
| 2xl | 48px | N/A in this phase |
| 3xl | 64px | N/A in this phase |

Exceptions:
- Node card internal padding: 12px top/bottom × 14px left/right (existing pattern — do not change)
- Icon box: 32×32px with 6px border-radius (existing pattern — do not change)
- ResourceNode fixed width: 168px (existing layout contract — do not change)
- DetailPanel fixed width: 320px (`w-80`) — do not change
- FilterPanel fixed width: 224px (`w-56`) — extend with "Source" filter section at same width

Source: extracted from `viewer/src/components/ResourceNode.tsx`, `DetailPanel.tsx`, `FilterPanel.tsx`.

### Phase 2 New Visual States

**Azure resource group containers** use VNet zone styling: same `vpc` zone color (`rgba(140,79,255,0.02)` background, `rgba(140,79,255,0.4)` border, `#a78bfa` label) — Azure VNet maps to VPC zone type.

**Shadow node** (SHD-02): dashed border `#d97706`, body opacity 1.0, "shadow" sub-label `#f59e0b`, estimated cost shown in grey mono text. Existing `isShadow` branch in `ResourceNode.tsx` already implements this — do not change visual contract.

**Policy finding source badge**: new `source: "policy"` field on Finding. In FindingCard, render a small violet pill `POLICY` beside the severity badge using `background: rgba(167,139,250,0.15)`, `color: #a78bfa`, `border: 0.5px solid rgba(167,139,250,0.3)`. Security rule findings have no source pill (backwards compatible).

**Compliance framework tags** (SEC-06): below each FindingCard body, render a row of grey mono pills for each `framework_ids` entry (e.g. `CIS`, `NIST`, `SOC2`, `PCI-DSS`). Style: `background: #1c2333`, `color: #4a5568`, `border: 1px solid #252d3d`, `font-size: 9px`, `padding: 1px 5px`, `border-radius: 3px`.

## Interaction Contract

### DetailPanel — Changes Tab (PLN-03, D-13, D-14)

The `Changes` tab is already implemented in `viewer/src/components/DetailPanel.tsx`. The Phase 2 contract confirms and locks this design:

- Tab order: `Overview | Findings (N) | Attributes | Changes (N)` — Changes tab only visible when `drift_changes.length > 0`
- Tab indicator: 2px bottom border in resource color (same as other tabs)
- Diff row layout: attribute name in mono 11px semibold `#e2e8f0` on top; before value in `#ef4444` with `text-decoration: line-through`; after value in `#22c55e` below
- Sensitive values: display `[sensitive]` in `#4a5568` italic — never reveal value
- Empty Changes tab: show GitCompare icon + "No changes" in `#22c55e` — existing implementation correct

No changes required to ChangesTab — it is implemented per D-13/D-14.

### FilterPanel — Source Filter (D-09, POL-01)

Add a new "Source" filter section to FilterPanel below the existing "Drift Status" section:

- Section header: `text-[10px] uppercase tracking-wider font-semibold` `color: #4a5568` — matches existing section headers
- Options: `Security` and `Policy` checkboxes
- Active state: `color: #a78bfa` for Policy (violet accent), `color: #e2e8f0` for Security
- Count badge: same `text-[10px] color: #374151` pattern as other filter counts
- Store: add `sources: string[]` to Zustand filter state; `toggleSourceFilter(source)` action
- Filtering behaviour: when Policy selected, show only nodes with at least one finding where `finding.source === 'policy'`; when Security selected, show nodes with findings where `finding.source === 'security'` (or source field absent — backwards compatible)

### Node Border — Drift Status (D-15)

Node border colour for drift states (confirmed from existing `driftColors` + D-15 decision):

| Drift Status | Border Colour | Border Style |
|-------------|--------------|-------------|
| added | `#22c55e` | solid |
| changed | `#f59e0b` | solid |
| deleted | `#ef4444` | solid + opacity 0.4 on entire node |
| unchanged | `#252d3d` | solid (default card border) |
| shadow | `#d97706` | dashed |

Node body and icon stay normal — border and opacity only. Source: D-15, existing `ResourceNode.tsx`.

### Shadow Badge Distinction (D-04, SHD-02)

The `shadow` sub-label below the node (`fontSize: 9, color: #f59e0b`) is visually distinct from finding severity badges:
- Finding badges: circular, filled background, in the node footer
- Shadow label: below the node card, text-only, amber, no background — intentionally minimal

### Upgrade CTA — Founding Member Gate (carries from Phase 1 D-01, D-02)

Applies to all new finding types (Azure + Policy + Shadow infra):

- In gated mode: finding count visible, details blurred (`filter: blur(4px)`)
- CTA link: `https://infracanvas.dev/founding` (existing `DetailPanel.tsx` href — do not change)
- CTA text: "Unlock details — founding member $49/mo" (existing — do not change)
- Sub-label: "Locked forever for founding members"
- Style: `background: #3b82f620`, `border: 1px solid #3b82f6`, `color: #60a5fa`, `border-radius: 6px`, `padding: 8px 16px`
