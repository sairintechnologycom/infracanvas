<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML rule files -> engine | Rule YAML loaded from package-internal directory (trusted) |
| Policy YAML -> engine | User-provided policy YAML from `--policy` flag directory (untrusted input) |
| .tf files -> parser | User's Terraform files parsed by python-hcl2 (untrusted input) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | security/loader.py `load_policy_rules()` | mitigate | Use `yaml.safe_load()` (already used in loader.py line 33) — never `yaml.load()`. Validate policy dir path is a real directory before rglob. |
| T-02-02 | Information Disclosure | main.py parse error output | mitigate | Only print filename (`path.name`) and exception message in warnings — never print full file contents or file paths outside the scanned directory. |
| T-02-03 | Denial of Service | parser/hcl.py | accept | Malformed HCL files could cause slow parsing via python-hcl2 Lark parser. Acceptable: CLI is local tool, not a service. Per-file error collection limits blast radius. |
</threat_model>
